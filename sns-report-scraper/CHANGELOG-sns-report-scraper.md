## 2026-07-21 ~ 2026-07-22 (V6.0 노션 연동 + 스크린샷 내보내기 — 대규모 변경)
- **로그인 안정성**
  - 트위터 로그인 시 정상 비밀번호도 거부되던 문제 수정(자동화 전용 브라우저를 봇으로
    의심해 차단하던 것이 원인)
    (기술: `browser-stealth.js`에 `launchLoginBrowser`/`closeLoginBrowser` 추가 —
    `chromium.launchPersistentContext(profileDir, {channel:'chrome', ...})`로 시스템 설치
    크롬을 전용 프로필로 실행, 크롬 미설치 시 기존 `chromium.launch()`로 자동 폴백)
  - 로그인 세션 저장 로직을 위 방식에 맞게 재작성, 플랫폼별 전용 크롬 프로필 폴더 분리
    (기술: `login-session.js`의 `PLATFORMS` 설정에 `profileDir: chrome-profile/{platform}`
    추가, `.gitignore`에 `chrome-profile/` 추가)

- **스크린샷 내보내기 (신규 기능)**
  - 리포트의 트위터/인스타 표를 각각 이미지 파일로 저장하는 버튼 추가
    (기술: `html2canvas` 라이브러리 추가, 캡처 직전 버튼/`.piechart` 요소
    `visibility:hidden` 처리 후 캡처, `canvas.toBlob()` + `URL.createObjectURL()`로 다운로드)

- **배포 안정성**
  - `update.bat`이 코드만 갱신하고 새로 추가된 라이브러리는 설치하지 않던 문제 수정
    (기술: robocopy 이후 `npm install` 단계 추가, 포터블/시스템 Node 감지 분기)

- **노션 연동 (신규 기능)**
  - 리포트를 노션 페이지에 자동으로 표로 전송하는 기능 추가
    (기술: `notion-export.js` 신설 — Notion REST API(`Notion-Version: 2022-06-28`) 호출,
    개인/팀 계정 단위 내부 통합 연결 필요)
  - 연동 설정 없이도 바로 쓸 수 있는 복사-붙여넣기용 마크다운 표 생성 기능 추가
    (기술: `buildMarkdownExport()`, 대시보드에 클립보드 복사 버튼 연결)
  - 표에 자사(PW)=파랑/경쟁사(BH)=빨강 배경색, 우세/경합/약세 결과 색상 추가
    (기술: rich_text `annotations.color`(`blue_background`/`red_background`/
    `green_background`/`yellow_background`) 활용)
  - PW/BH 수치 비율을 막대그래프 형태로 보여주는 셀 추가
    (기술: 블록 문자(`█`)를 두 색상으로 나눠 채우는 `buildBarCell()`, 비율이 0에
    가까워도 최소 1칸은 보이도록 안전장치)
  - 칸 이름을 이모지 1자로 압축해 칸 너비 부담 축소
    (기술: `FIELD_LABELS`를 텍스트에서 이모지로 교체)
  - 전송할 때마다 새 페이지를 만드는 대신, 처음 한 번만 페이지를 만들고 이후엔 같은
    페이지를 갱신하는 방식으로 전환 — 노션에서 손으로 맞춘 칸 너비가 유지되도록
    (기술: 최초 생성 시 `GET /blocks/{id}/children` 재조회로 블록 ID 확보 후
    `notion-config.json`에 저장, 이후엔 표 블록은 유지한 채 `table_row` 자식만
    `DELETE`+`PATCH`로 교체 — 실제 API 재검증은 다음 실사용 확인 예정)

- **기타**
  - 노션의 "진짜 트윗 카드" 임베드(2칸 나열 등)는 노션이 화면에서 직접 붙여넣을 때만

## 2026-07-20 (V5.0 설치형 배포 패키지 전환 + 안정성 버그 수정 — 대규모 변경)
- **설치형 배포 패키지 (신규 기능)**
  - 압축 파일 수동 배포 대신, 더블클릭 몇 번으로 끝나는 설치 마법사 방식으로 배포 전환
    (기술: `installer.iss` 신설 — Inno Setup 기반, `{localappdata}` 하위에 설치해 관리자 권한 불필요, 기존 포터블 폴더 구조(Node·Playwright 내장) 그대로 패키징)
  - 바탕화면·시작 메뉴에 실행/업데이트 바로가기 자동 생성
    (기술: `[Icons]` 섹션에 `start-dashboard.bat`/`update.bat` 각각 등록)
  - 개인 로그인 세션·리포트·내부 문서는 설치 패키지에서 자동 제외
    (기술: `[Files] Excludes`에 `*-session.json`, `\reports\*`, `*.md`, `*.iss` 등 명시)

- **로그인 세션 저장 안정성**
  - 로그인 완료 후 대시보드 버튼이 영원히 잠긴 채로 남던 문제 수정
    (기술: `login-session.js`의 `browser.close()`가 드물게 반환되지 않고 멈추는 문제 — `Promise.race`로 타임아웃 처리 후 `process.exit(0)` 강제 종료)
  - 화면 재접속 시 이미 끝난 작업인데도 버튼이 잠긴 채 남던 문제 수정
    (기술: SSE `init` 이벤트 핸들러에도 `status !== 'running'`일 때 버튼 잠금 해제 로직 추가, `refreshStatus()`에서 진행 중 작업 감지 시 `startEventStream()` 재호출)

- **매칭 정확도 버그 수정**
  - 경쟁사 계정이 자기 게시물을 반복 재게시하는 경우 항상 누락되던 버그 수정(실측 사례: 헌터x헌터)
    (기술: `socialContext` 라벨이 `/재게시함$/`이면서 게시물 링크의 작성자가 수집 대상 계정과 같으면 필터링 예외 처리)

- **UI 개선**
  - 기간 지정 영역 접기/펼치기 제거, 항상 펼친 상태로 고정
  - 시작일 선택 시 종료일에 동일 날짜 자동 입력
    (기술: 시작일 `input[date]`의 `onchange`에서 종료일 필드 값 동기화)

- **배포 안내 페이지 개편**
  - 허브 페이지 카드의 설치 안내를 전용 다운로드 페이지(`sns-report-download.html`)로 분리
  - 다운로드/대시보드 열기 버튼을 2열 배치로 재구성, 로그인 정보 미수집·설치 시 방화벽 팝업 안내 문구 추가
  - 압축 해제 안내를 설치 마법사·바탕화면 아이콘 안내로 교체, 다운로드 링크를 배포 저장소의 설치 파일 직접 연결로 교체

- **배포 저장소 분리**
  - 팀 공유용 릴리스를 내부 개발 문서가 없는 별도 저장소로 분리해서 배포
    (기술: 개발은 기존 저장소에서 계속하되, 배포용 zip/설치 파일만 별도 저장소의 릴리스에 업로드)

## 2026-07-16 (V4.0 실행 명령어 정리 + 리포트 내장 편집 기능 — 대규모 변경)
- **실행 명령어 정리**
  - `run.js`를 `run-megahouse.js`로 이름 변경 — 임의 계정 단독 수집(`collect-account.js`)과 혼동 방지
    (기술: 관련 파일 전체 참조 갱신, 기능 변경 없음)
  - 날짜 두 개 대신 "today" 한 단어로 오늘 하루 지정 가능하도록 추가
    (기술: `Intl.DateTimeFormat`에 `Asia/Seoul` 타임존을 명시해 서버 시간대와 무관하게 KST 기준으로 계산)
  - 날짜를 아예 안 주고 실행하면 옛날 기본값으로 조용히 넘어가던 것을 막고 에러 후 종료하도록 변경
    (기술: 죽은 기본 설정값(`CONFIG.startDate`/`endDate`) 제거, 날짜 인자 필수화)
  - 예전부터 있던 "종료일 미포함" 안내 문구가 실제 동작(포함)과 반대였던 문서 오류 수정

- **수집 안정성**
  - 링크가 여러 개 붙어 화면에서 유난히 긴 게시물 하나가 스크롤 중 통째로 누락되던 사례 발견 → 스크롤 폭 축소로 완화
    (기술: 페이지 스크롤 단위를 3000px → 1500px로 줄여 스냅샷 간 겹치는 구간 확대)
  - 게시물이 필터에 걸려 제외될 때마다 사유를 기록하는 진단 로그 추가
    (기술: `socialContext`/인용 중첩 판정 결과를 콘솔에 출력 — 이후 원인 불명 누락 재발 시 즉시 원인 확인 가능)
  - SNS 실적 수집 시 재고 스냅샷도 매번 자동으로 함께 촬영하도록 연결
    (기술: `run-megahouse.js`에서 `naver-stock-snapshot.js`의 `captureSnapshot()` 호출, 실패해도 SNS 리포트 저장은 계속되도록 예외 격리)

- **매칭 정확도 버그 수정**
  - 자사가 여러 프랜차이즈 발표 게시물마다 재사용하는 "초회특전 SET(방석)" 문구 때문에 완전히 다른 상품끼리 잘못 묶이던 버그 수정(실측 사례: 헌터x헌터 ↔ 페르소나3)
    (기술: `GENERIC_KEYWORDS`에 '초회'/'방석'/'SET' 추가)
  - 곱셈 기호(×)와 영문자(x)가 다른 토큰으로 갈라져 같은 프랜차이즈명인데도 매칭이 아예 안 되던 버그 수정
    (기술: `extractKeywords`에서 `×` → `x` 정규화 단계 추가)
  - 상품명/키워드 추출 로직을 별도 파일로 분리(다음 항목의 전제 작업)
    (기술: `matching-core.js` 신설 — `aggregate.js`와 리포트 내장 브라우저 스크립트가 완전히 같은 로직을 공유하도록 함, Node 전용 API 없이 순수 함수만 구성)

- **리포트 내장 편집 기능 (신규 기능)**
  - 미매칭 게시물을 채팅으로 알려주지 않고 리포트 화면에서 직접 표에 추가하는 기능 추가
    (기술: 미매칭 목록 각 항목에 "표에 추가" 버튼, 이미 아는 지표값 그대로 사용)
  - 게시물 상세페이지를 통째로 복사한 텍스트를 붙여넣으면 본문·게시 시각·좋아요·리트윗을 자동으로 읽어 추가하는 기능 추가(자동 인식 실패 시 직접 입력 대체 경로 포함)
    (기술: `paste-parser.js` 신설 — "조회수" 다음에 오는 4개 숫자(답글/RT/좋아요/북마크)를 위치 기반으로 파싱, 오전/오후 12시간제 KST 시각을 ISO로 변환)
  - 추가한 게시물을 실제 파이프라인에 영구 반영하는 저장 구조 추가
    (기술: `manual-posts.json` + `aggregate.js`의 `applyManualPosts()` — "(수동 추가)" 계정으로 합쳐진 뒤 평소 자동 매칭이 그대로 재적용됨. `run-megahouse.js`/`rebuild-report.js` 양쪽에 연결)
  - 실수로 추가한 항목을 저장 전에 없앨 수 있는 삭제 버튼 추가
    (기술: 삭제 시 화면의 행 + 저장용 배열(`window._manualAdditions`) 양쪽에서 제거)

## 2026-07-10 (V3.0 총판매추정 안정화 + PW·BH 종합 비교표 — 대규모 변경)
- **총판매추정 계산 방식 수정**
  - "우리가 이 상품을 처음 관측한 시점" 기준으로 계산하던 걸, 실제 과거 기록 유무와 무관하게 **항상** 초기 판매한도 역산 방식으로 통일
    (기술: `hasRealBaseline` 분기 제거 — 재고 추적을 시작한 지 며칠 안 된 상품은 실제 판매량과 무관하게 총판매추정치가 작게 나오던 문제 해결)
  - 재고 스냅샷 표에 "직전 스냅샷 대비" 컬럼 추가(총판매추정과 별개로, 순수 실측 변화량만 보여주는 참고용 컬럼)

- **매출 비교 시각화 통일**
  - SNS 표의 매출 매칭 칸을 리트윗/좋아요와 같은 "PW vs BH 분할 바" 형식으로 통일
    (기술: salesBar 컴포넌트 — 기존 순위+개수 텍스트 두 칸을 분할 바 한 칸으로 병합)
  - 매출 칸에도 리트윗/좋아요처럼 회색 작은 글씨로 PW:BH 점유율 캡션 추가
    (기술: "77:23" 형식, 두 값 합산 기준 백분율)
  - 매출 매칭 칸의 순위(초록)·개수(진회색) 색상 분리 및 좌측 정렬로 가독성 개선
  - 표 너비가 브라우저 폭을 살짝 넘던 문제 → 매출 칸 전용 CSS로 폭 축소

- **PW·BH 종합표 (신규 기능)**
  - 재고 스냅샷 섹션에 PW/BH 재고 상품을 자동으로 짝짓는 "종합표"를 최상단에 신규 추가(기존 PW표·BH표는 그대로 유지)
    (기술: `matchPwBhStockProducts` — SNS 매칭과 같은 키워드 겹침 규칙을 재고 상품명에도 재사용)

    (기술: 총판매추정 값 기준 SVG 라인차트, 스냅샷 3개 미만이면 안내 문구로 대체, 날짜/시각을 2줄로 나눠 표시)

- **버그 수정**
  - 종합표가 매출순 정렬 없이 매칭된 순서 그대로 나오던 정렬 누락 버그 수정

    (기술: 최초 시점=100 지수화를 시도했다가, 실측 데이터로 재현 확인 후 최종적으로 PW/BH가 각자 축을 가진 그래프 2개로 분리 — 각 축은 그 시리즈 자체의 최소~최대 범위로 확대하고 0을 강제하지 않음)

    (기술: Union-Find의 전이적 그룹화가 원인 — 같은 프랜차이즈의 단품/세트 변형 상품이 프랜차이즈명만으로 한 그룹에 뒤섞여 통째로 버려짐. 그룹화 대신 PW-BH 쌍마다 자카드 유사도 점수를 매기고 "상호 최선"일 때만 확정하는 방식으로 교체 — 실측 기준 14쌍 → 27쌍으로 개선)
