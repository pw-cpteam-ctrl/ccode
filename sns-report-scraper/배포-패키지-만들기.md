# 배포 패키지 만들기 (배포 담당자용, 한 번만 하면 됨)

> 이 문서는 **팀에 나눠줄 "포터블 패키지"를 만드는 사람**을 위한 것입니다.
> 팀원은 이 문서를 볼 필요 없이 `팀원-사용법.md`만 보면 됩니다.
>
> 포터블 패키지 = Node·크로미움 브라우저·라이브러리까지 폴더 하나에 전부 넣은 것.
> 받는 사람은 아무것도 설치 안 하고, 압축 풀고 더블클릭만 하면 됩니다.
>
> ⚠️ 이 작업은 **윈도우 PC**에서 해야 합니다(팀원이 윈도우를 쓰니까).

---

## 준비물 (처음 딱 한 번)

### 1. 포터블 Node.js 넣기
1. https://nodejs.org → 상단 **Downloads** → **Windows Binary (.zip)** 의 **64-bit** 받기
   (설치판 .msi 말고 **.zip** 이어야 함. 예: `node-v20.xx.x-win-x64.zip`)
2. 압축을 풀면 `node-v20.xx.x-win-x64` 폴더가 나옴.
3. 그 폴더 **안의 내용물 전부**를 이 `sns-report-scraper` 폴더 밑에 `node` 라는 이름의 폴더로 넣기.
   → 최종적으로 `sns-report-scraper\node\node.exe` 가 있어야 함.

### 2. 라이브러리 + 브라우저 넣기
1. cmd(명령 프롬프트)를 열고 이 폴더로 이동:
   ```
   cd /d 여기에\sns-report-scraper\경로
   ```
2. 아래 3줄을 순서대로 실행:
   ```
   set PLAYWRIGHT_BROWSERS_PATH=0
   node\npm.cmd install
   node\npx.cmd playwright install chromium
   ```
   - `set PLAYWRIGHT_BROWSERS_PATH=0` 은 **크로미움 브라우저를 `node_modules` 안에 설치**하라는 뜻입니다.
     이래야 패키지를 통째로 옮겨도 브라우저가 같이 따라가요(이게 포터블의 핵심).
   - 이 창을 닫지 말고 세 명령이 다 끝날 때까지 두세요(브라우저 다운로드에 몇 분 걸림).

### 3. 잘 되는지 확인
- `start-dashboard.bat` 을 더블클릭 → 대시보드가 열리고, 로그인 버튼을 누르면 **브라우저 창이 실제로 뜨는지** 확인.
- 여기까지 되면 패키지 재료가 다 준비된 것.

---

## 압축해서 배포하기 (버전 올릴 때마다)

### 4. 개인 정보 빼고 압축
압축 **전에** 아래를 지우거나 빼세요(본인 로그인/데이터라 팀원에게 넘기면 안 됨):
- `x-session.json`, `instagram-session.json` (내 로그인 세션)
- `chrome-profile` 폴더 (로그인용 전용 크롬 프로필 — 내 로그인 정보가 그대로 들어있음)
- `reports` 폴더 안의 내용 (내가 뽑은 리포트)

그다음 `sns-report-scraper` 폴더를 **통째로** ZIP으로 압축.
- 이때 `node`, `node_modules` 폴더도 **포함**되어야 함(용량 200~300MB 정도가 정상).
- 파일 이름 예: `sns-report-v2.3.zip`

### 5. 깃허브 릴리스에 올리기
1. 깃허브 저장소 페이지 → 오른쪽 **Releases** → **Draft a new release**
2. **Tag** 에 버전 적기(예: `v2.3`)
3. 방금 만든 ZIP을 **첨부 파일 영역에 드래그**해서 업로드
4. **Publish release**

이러면 팀원은 허브 페이지의 "다운로드" 버튼으로 이 ZIP을 받게 됩니다.
(허브 페이지 버튼 연결은 첫 릴리스가 올라온 뒤에 한 번 걸어두면 됩니다.)

---

## 그 뒤로 업데이트는 훨씬 간단함

- **소스 코드만 고쳤을 때(제일 흔함):** 릴리스 다시 안 만들어도 됩니다.
  그냥 평소처럼 브랜치에 커밋+푸시만 하면, 팀원은 `update.bat` 더블클릭으로 바뀐 부분만 받아갑니다.
- **무거운 게 바뀌었을 때(라이브러리 추가·Playwright 버전업 등, 드묾):**
  위 4~5번을 다시 해서 새 버전(`v2.4.zip`)을 릴리스에 올리고, 팀원은 새 ZIP을 한 번 다시 받습니다.
