- **어떻게 처리했나**: 뱃지 크기를 슬라이더 값이 아니라 "실제로 화면에 표시되는 이름 크기"에 비례하도록 계산해서 직접 지정

### 다운로드-미리보기 미묘한 차이(뱃지 밀림·스토어 정렬)의 진짜 근본 원인 해결
- **무슨 문제/요청이 있었나**: 뱃지 텍스트 위치가 미묘하게 밀리고, 오픈헤더 스토어명이 중앙정렬 안 되고 밀려 보인다는 신고. "익스포트랑 미리보기가 왜 자꾸 미묘하게 차이나냐"는 반복 지적과 함께, 페르소나처럼 이름이 길어 확대하면 뱃지가 커지기만 하고 위로 안 올라간다는 지적
- **확인해보니**: 지금까지 여러 차례(1~8차) 여유 마진으로 증상만 땜질해왔는데, 진짜 근본 원인은 따로 있었음 — 다운로드 엔진(html-to-image)이 CORS 때문에 CDN 폰트(프리텐다드·페이퍼로지)를 못 읽어서, 다운로드 시엔 화면과 다른 폴백폰트로 그려지고 있었던 것. 그래서 화면과 다운로드가 항상 미세하게 다른 폭으로 렌더됨
- **어떻게 처리했나**: 다운로드 직전 웹폰트를 CDN에서 받아 data URL로 만들어 `htmlToImage`의 `fontEmbedCSS` 옵션으로 직접 박아넣음 → 화면과 다운로드가 100% 동일한 폰트로 렌더. 이후 그동안 넣어뒀던 여유 마진들을 전부 제거하고, 배지 배치 로직도 "이름이 슬라이더 크기 그대로 옆에 들어가면 옆 유지, 안 들어가면 위로"로 정확하게 재정리. 배지 위치를 조정할 때마다 처음부터 다시 판단하도록 해서, 한 번 위로 올라간 배지가 조건이 바뀌어도 안 내려오던 부수 버그도 같이 잡음
- **결과**: 사용자 실제 데이터로 화면·다운로드 결과물을 나란히 비교 검증, 뱃지 밀림·스토어 정렬·잘림 모두 해소되고 페르소나 확대 시 배지가 정확히 위로 올라가는 것까지 확인 후 `main` 반영. 사용자가 "이제야 되네"로 확인. **CHANGELOG에 9차로 근본원인·해결법·교훈("여유 마진으로 덮기 전에 근본을 없애라") 상세 기록**
