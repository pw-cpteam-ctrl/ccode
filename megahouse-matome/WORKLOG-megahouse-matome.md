### 로컬 git pull이 계속 실패하던 문제 (잘못된 파일명)
- **무슨 문제/요청이 있었나**: 수정을 push했는데도 사용자 PC에 반영이 안 됨. 로컬 index 파일 날짜가 옛날 그대로였음
- **확인해보니**: 이전에 실수로 파일명 자체에 윈도우 경로(`C:\Users\Brain\ccode\CHANGELOG-작성-규칙.md`)가 통째로 들어간 파일이 커밋돼 있었고, 윈도우는 파일명에 `\`·`:`를 못 써서 `git pull`이 `invalid path`로 매번 중단되고 있었음. 그래서 사용자 로컬이 한 번도 최신화되지 못한 것
- **어떻게 처리했나**: 그 잘못된 파일을 저장소에서 삭제(`git rm`)하고 push. 이후 `git fetch` + `git reset --hard origin/main`으로 로컬을 강제로 최신과 일치시키도록 안내
- **결과**: pull 정상화, 로컬 파일 최신화 확인

### 다운로드 이미지에서만 글자가 잘리는 버그 (미리보기는 정상)
- **무슨 문제/요청이 있었나**: 다운로드한 PNG에서 제품명(리로드·코난 등)과 오픈헤더 스토어명 끝 글자가 잘리고 뱃지와 겹친다는 신고. "미리보기랑 실제 다운로드 결과가 다르다"는 지적이 반복됨
- **확인해보니**: 다운로드 엔진(html-to-image)이 텍스트를 화면보다 미세하게 넓게 그리는데, 글자 상자가 내용에 딱 맞게 `overflow:hidden`으로 잘려 있어 끝 글자가 잘려나가는 것이 원인. 처음 다섯 번은 테스트 환경에 실제 폰트가 없어 재현이 안 돼 헛다리를 짚었고, 사용자 실제 데이터 + 프리텐다드/페이퍼로지 폰트를 강제로 심어 재현한 뒤에야 원인을 눈으로 확정함(상세 시도 기록은 CHANGELOG 참고)
- **어떻게 처리했나**: 강제 잘림 제거 + 긴 이름은 뱃지를 윗줄로 이동 + 글자 크기 자동 축소에 여유 마진 부여 + 폰트 로드 후 재조정. 실제 폰트 환경에서 다운로드 결과물 자체를 미리보기와 나란히 비교해 검증
- **결과**: 미리보기·다운로드 결과가 일치하고 모두 안 잘리는 것을 브라우저로 확인 후 `main` 반영. **재발 방지 교훈을 CHANGELOG에 교과서 형태로 상세 정리**
