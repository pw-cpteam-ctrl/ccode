# CHANGELOG — 입고안내 이미지 자동 제작 툴 (inbound-image-composer)

---

## 2026-07-09 (V1.0 ~ V1.7 최초 구현부터 분위기 클러스터 자동분류까지)

- **최초 구현 (V1.0)**
  - 6단계 플로우(업로드/그리드검출 → 표정리 → 전체미리보기 → 삭제확인 → 순서정렬 →
    최종분할/내보내기)를 화면 전환으로 분리한 바닐라 JS + Canvas 웹 앱으로 신규 제작
  - `reference/grid-detect.js`, `reference/render-page.js`의 알고리즘·수치를 그대로 포팅
  - 삭제는 "체크박스 선택 → 확인 목록 표시 → 반영" 2단계로 구현해 누락/오삭제 방지
  - IP명 사전·등급표(S/A)·분위기 클러스터·스토어 프로필을 UI("⚙ 사전 관리")에서 편집
    가능하게 하고, insta-gen과 동일한 패턴으로 GitHub 데이터 브랜치 커밋 지원
  - Playwright로 6단계 전체 플로우 실제 조작 검증 완료

- **저장 방식 개선 (V1.1)**
  - 여러 프로젝트를 동시 운영하며 백엔드 비용 부담을 줄이기 위해, "이 브라우저에 저장"
    (localStorage)을 기본 동작으로 전환 — GitHub 커밋은 팀 공유가 필요할 때만 쓰는
    선택 기능으로 낮춤
  - 백엔드 미배포 상태(`index.html`만 열어도)에서도 사전 편집 내용이 새로고침 후
    유지되도록 변경, GitHub 공유 버튼도 백엔드 없으면 에러 없이 "로컬에만 저장됨" 안내로 처리

- **AI 자동 채우기 도입 (V1.2)**
  - 1단계 크롭 확정 후 "🤖 AI로 채우기"를 누르면 원본 스크린샷을 Claude Vision
    (`claude-haiku-4-5`)으로 분석해 2단계 표(IP명/가격/태그/배송비)를 자동으로 채우는
    신규 기능 추가 — 대화형 AI로 하던 경험을 웹 도구에 그대로 재현
  - structured output(`json_schema`)으로 항목 개수를 그리드 셀 수에 정확히 맞춰 수신,
    IP명 사전 일부를 프롬프트에 포함해 표기 일관성 확보
  - business-rules.md의 "애매하면 확인" 원칙에 따라 AI 결과는 항상 `aiUncertain` 플래그
    + "⚠ AI 추정 - 확인 필요" 배지로만 표시, 사람이 직접 수정해야 확정되도록 제한
  - `ANTHROPIC_API_KEY` 미설정(백엔드 미배포) 시에도 에러 없이 "정상입니다" 안내만 뜨고
    수동 입력으로 계속 진행 가능하게 처리

- **2단계 UI 개편 + AI 안정성 보강 (V1.3)**
  - 버그 수정: AI로 채우기가 `ip` 필드를 빈 문자열로 돌려주는 문제 발견 → 프롬프트에
    "확신 없어도 rawText를 ip에 복사해 절대 비우지 말라" 규칙 추가, 프론트에도
    `item.ip = result.ip || result.rawText` 안전장치 추가
  - 2단계 표를 한 줄짜리 `<table>`에서 사진당 IP명/태그/가격/등급 카드 2열 그리드로 재구성
  - 거의 항상 무료배송이라 편집 불필요했던 배송비 입력칸을 화면에서 제거(데이터는
    `item.ship`에 남아있고 기본값 무료배송 유지, 예외 변경은 3단계 카드 수정 다이얼로그에서 가능)
  - 상품 수가 많을 때 목록이 지나치게 길어지던 가독성 문제 해결

- **AI 인식률 개선 (V1.4)**
  - 버그 수정: 실사용 중 25개 상품을 이미지 1장으로 통째로 보내면 인식률이 1/5 수준으로
    떨어지는 문제 확인 — 항목이 많을수록 항목당 AI가 쓸 수 있는 시각 해상도/주의력이
    쪼개지는 게 원인
  - 사진은 빼고 상품명/가격 텍스트 영역만 크롭·2배 확대해 6개씩 세로로 이어붙인 이미지로
    나눠서 배치 호출하도록 변경 (예: 25개 → 5번 호출) → 항목당 정보량 4~5배 증가로 인식률 개선
  - `api/parse-image.js`에 `layout: 'textStrip'` 모드 추가, 번호 매긴 텍스트 블록을
    순서대로 읽는 전용 프롬프트 분리

- **배포 안정화 (V1.5)**
  - 버그 수정: 모노레포 특성상 insta-gen 등 무관한 프로젝트 커밋에도 이 Vercel 프로젝트가
    매번 재배포를 시도해 Hobby 요금제 하루 100회 배포 한도를 다른 도구 때문에 소진하는
    문제 발생 → `vercel.json`에 `ignoreCommand` 추가해 이 폴더를 건드리지 않은 커밋은
    빌드를 건너뛰도록 수정
  - 버그 수정: `ignoreCommand`가 Root Directory(=`inbound-image-composer`) 안에서 실행됨을
    간과하고 폴더명을 경로로 그대로 써서 존재하지 않는 하위 폴더를 찾게 되어 git diff가
    항상 "변경 없음"으로 나와 이후 모든 배포가 영구히 스킵되는 심각한 버그 발생 →
    같은 레포의 insta-gen/product-sns-formatter와 동일한 `"./"` 패턴으로 통일해 해결

- **문서화 & 모바일 반응형 (V1.6)**
  - 프로젝트 전체 아키텍처·데이터 흐름·기술 결정 배경을 정리한 `PLAN.md` 신규 작성
  - `api/parse-image.js`에 배치 인식 전략·프롬프트 설계·에러 처리 흐름을 설명하는
    JSDoc 주석 보강
  - 1024px/768px/480px 반응형 브레이크포인트 추가 — 헤더/네비게이션/다이얼로그/카드
    레이아웃이 화면 크기에 따라 재배치되도록 개선, 버튼 터치 영역 최소 40px로 확대
  - 메인 툴킷(`/index.html`)에 이 도구의 바로가기 카드 추가 (배포된 Vercel 도메인
    `https://inbound-image-composer.vercel.app/`로 연결)

- **분위기 클러스터 자동분류 (V1.7)**
  - "AI로 채우기"가 IP명/가격/배송비/태그뿐 아니라 5단계 자동 정렬에 쓰이는 **분위기
    클러스터까지 함께 판단**하도록 확장 — 기존 클러스터 사전을 프롬프트에 함께 전달해
    신규 IP가 어느 클러스터와 어울리는지 AI가 고르게 함
  - `app.js`에 `applyAiClusterResult()` 추가 — `uncertain=false`로 확신 있게 읽은 IP만
    실제로 클러스터 사전에 편입, 애매한 추정으로 5단계 정렬 기준(클러스터 사전)이
    오염되는 것을 방지. 편입이 발생하면 즉시 localStorage에 저장해 다음 인식부터 재사용
  - 2단계 표 정리 그리드를 2열 → 3열로 변경(카드가 화면에 비해 너무 넓었던 문제 해결),
    3·5단계 미리보기 그리드를 5열 → 2열로 변경(카드/사진이 너무 작게 보이던 문제 해결)
