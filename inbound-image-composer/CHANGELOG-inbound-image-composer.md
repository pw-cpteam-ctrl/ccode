# CHANGELOG — 입고안내 이미지 자동 제작 툴 (inbound-image-composer)

---

## 2026-07-09 (V1.0 ~ V1.6 최초 구현부터 모바일 대응까지)

- **최초 구현 (신규 기능)**
  - 업로드 → 표정리 → 미리보기 → 삭제확인 → 순서정렬 → 최종내보내기, 6단계 화면으로 분리한 웹 도구 최초 제작
    (기술: 바닐라 JS + Canvas, 화면 전환 방식으로 단계 분리, reference/grid-detect.js·render-page.js 알고리즘 그대로 포팅)
  - 삭제는 체크박스 선택 후 확인 목록을 한 번 더 보여준 뒤에만 반영
    (기술: 2단계 확인 플로우, 실수로 누락되는 것 방지)
  - IP명 사전·등급표·분위기 클러스터·스토어 프로필을 화면에서 직접 편집 가능
    (기술: "⚙ 사전 관리" 다이얼로그, insta-gen과 동일한 GitHub 데이터 브랜치 커밋 패턴)

- **저장 기능 개선**
  - 배포 없이 브라우저만 열어도 사전 편집 내용이 유지됨
    (기술: localStorage 기본 저장으로 전환, GitHub 커밋은 팀 공유 시에만 쓰는 선택 기능으로 격하)

- **AI 자동 채우기 (신규 기능)**
  - 원본 스크린샷 크롭 후 버튼 한 번으로 IP명/가격/태그/배송비 자동 인식
    (기술: Claude Vision claude-haiku-4-5, JSON Schema structured output으로 항목 개수 강제)
  - AI가 채운 값은 항상 "확인 필요" 표시로만 들어가고 사람이 확정해야 함
    (기술: aiUncertain 플래그, business-rules.md의 "애매하면 확인" 원칙 적용)
  - 백엔드 미배포 시에도 에러 없이 수동 입력으로 계속 진행 가능
    (기술: ANTHROPIC_API_KEY 부재 시 500 응답을 프론트가 안내 메시지로 흡수)

- **안정성 개선**
  - AI 인식 결과의 IP명이 빈 칸으로 남는 문제 수정
    (기술: 프롬프트에 "확신 없어도 rawText를 ip에 복사" 규칙 추가, 프론트에도 폴백 로직 추가)
  - 2단계 표 정리 화면을 한 줄짜리 표에서 카드형 2열 그리드로 재구성
    (기술: `<table>` → CSS Grid 카드 레이아웃 변경)
  - 자주 안 쓰는 배송비 입력칸을 화면에서 제거해 목록 가독성 개선
    (기술: 데이터는 item.ship에 유지, 기본값 무료배송, 예외는 수정 다이얼로그에서 처리)

- **AI 인식률 개선**
  - 상품이 많을 때(25개 등) 인식률이 크게 떨어지던 문제 수정
    (기술: 사진 통째로 보내지 않고 텍스트 영역만 크롭·확대해 6개씩 배치 호출, layout: 'textStrip' 모드 추가)

- **배포 안정화**
  - 무관한 다른 프로젝트 커밋에도 이 도구가 매번 재배포되던 문제 수정
    (기술: vercel.json ignoreCommand 추가, 이 폴더 변경 여부로 빌드 스킵 판단)
  - ignoreCommand 경로 설정 오류로 이후 모든 배포가 영구히 스킵되던 심각한 버그 수정
    (기술: Root Directory 안에서 실행됨을 반영해 경로를 "./"로 통일)

- **문서화 & 모바일 대응**
  - 프로젝트 구조와 기술 결정을 정리한 설계 문서 신규 작성
    (기술: PLAN.md — 아키텍처, 데이터 흐름, 배포 체크리스트 포함)
  - 좁은 화면에서도 레이아웃이 깨지지 않도록 개선
    (기술: 1024/768/480px 반응형 브레이크포인트, 버튼 터치 영역 최소 40px 확보)
  - 메인 툴킷 페이지에 이 도구 바로가기 카드 추가
    (기술: 배포된 Vercel 도메인 링크 연결)

## 2026-07-10 (V1.7 ~ V1.8 클러스터 자동분류, 미리보기 재구성, 삭제 UX 개선)

- **분위기 클러스터 자동분류 (신규 기능)**
  - AI로 채우기가 IP명뿐 아니라 5단계 정렬에 쓰이는 분위기 클러스터까지 함께 판단
    (기술: 기존 클러스터 사전을 프롬프트에 포함해 신규 IP의 소속 클러스터 추정)
  - 확신 있게 읽은 항목만 클러스터 사전에 실제로 편입되도록 제한
    (기술: `applyAiClusterResult()`, uncertain=false인 경우에만 반영해 정렬 기준 오염 방지)

- **미리보기 레이아웃 재구성**
  - 미리보기 카드가 화면 폭에 비해 너무 좁거나 커지던 문제를 페이지 박스 단위로 재구성해 해결
    (기술: 5열×5줄(25개) 단위 박스로 묶고, 박스 자체를 2열 그리드로 나란히 배치)
  - 항목이 적어 박스 하나만 채워져도 카드가 늘어나지 않도록 빈 박스 틀 유지
    (기술: 박스 개수가 홀수면 빈 박스를 하나 더 추가해 항상 2열 짝을 맞춤, CSS grid 기본 stretch로 높이 자동 일치)
  - 2단계 표 정리 화면 카드 폭 조정
    (기술: 2열 → 3열 그리드로 변경)

- **여러 카드 묶어서 이동 (신규 기능)**
  - 카드 없는 빈 배경을 드래그하면 사각형 영역으로 여러 카드를 한 번에 선택 가능
    (기술: 마우스 좌표 기반 마퀴 선택, 겹치는 카드에 group-selected 클래스 적용)
  - 선택된 카드 중 하나를 드래그하면 전체가 순서를 유지한 채 함께 이동
    (기술: 배열 인덱스가 아닌 항목 id 기준으로 재배치를 계산해 그룹 이동 시에도 정확)

- **삭제 확인 단계 제거, 즉시 삭제 + 실행취소로 변경**
  - 삭제 시 별도 확인 화면을 거치지 않고 바로 반영
    (기술: 확인 단계를 없애 6단계 → 5단계로 압축)
  - 실수로 삭제해도 바로 되돌릴 수 있는 실행취소 배너 추가
    (기술: 삭제 직전 원래 배열 인덱스를 기록해 동일 위치로 재삽입, 가장 최근 삭제 1건만 지원)

## 2026-07-13 (V1.9 그리드 검출 버그 수정, 3·4단계 통합, 사전 병합 버그 수정, 성향 자동분류)

- **그리드 검출 버그 수정**
  - 입력 이미지 가로폭이 1000px가 아니면 그리드를 아예 못 잡던 문제 수정
    (기술: 절대 픽셀 좌표였던 sampleCols/cardW/cardH/임계값을 img.width 비례 스케일링으로 전환)
  - 5열인데 특정 열 하나가 통째로 안 잡히던 문제 수정 (예: 흰 배경 아이콘류 상품 사진이
    있으면 그 열이 계속 누락됨)
    (기술: `detectColStarts`가 딱 한 지점(첫 행 +40px)만 샘플링해서 하필 그 위치 색이
    흰색에 가까우면 열 전체를 놓치던 문제 — 행 탐지처럼 여러 행 × 여러 세로 위치를
    교차 샘플링해서 가장 많은 열이 잡힌 결과를 채택하도록 개선)

- **사전 병합 버그 수정**
  - 등급표(S/A급)에 이미 있는 IP(예: 주술회전)가 AI 채우기 후에도 "기타"로 분류되던 문제 수정
    (기술: `loadDict()`가 로컬 저장값이 하나라도 있으면 시드 전체를 통째로 덮어쓰던 버그 → gradeTable/moodClusters/storeProfiles/productLineNames 전부 합집합 병합으로 전환)

- **AI 성향 자동분류 (신규 기능)**
  - 등급표에 없는 IP만 AI가 남성향/여성향을 추측해 등급 드롭다운을 미리 채움 (사전 우선, 없는 것만 AI 보완)
    (기술: RESULT_SCHEMA에 genderLean 필드 추가, `applyAiGenderLean()`이 gradeTable S/A에 없는 IP에 한해서만 B급 남/여성향으로 채움 — B/C 등급 자체는 영업 판단이라 항상 B로 채우고 필요시 사람이 조정)

- **3·4단계 통합**
  - "전체 미리보기"와 "순서 정렬" 두 화면이 카드 드래그 이동 기능을 중복 지원하던 문제를 하나로 통합 (5단계 → 4단계로 축소)
    (기술: previewGrid/sortPreviewGrid 이중 렌더 제거, 자동정렬 버튼+스토어 선택을 미리보기 화면에 병합)

- **헤더 미리보기 (신규 기능)**
  - 최종 내보내기 화면에서 헤더 드롭다운을 바꾸면 실제로 어떤 이미지가 쓰이는지 바로
    아래에 썸네일로 확인 가능
    (기술: `updateHeaderPreview()`, 헤더가 비어 보이면 원본 스크린샷 자체가 1행 시작
    지점 위쪽에 배너를 포함하지 않고 캡처된 경우라 원본 캡처 범위를 다시 확인해야 함)

- **AI 크롭 확인 UX 개선**
  - 별도 "AI 전송 미리보기" 버튼을 제거하고 1단계 디버그 오버레이에 흡수
    (기술: 사진 크롭 범위(빨간 박스)와 AI에게 실제 전송되는 텍스트 포함 범위(초록 점선)를 같은 캔버스에 함께 표시)

- **2단계 편의 기능 (신규 기능)**
  - 같은 IP명끼리 카드 왼쪽 테두리 색으로 묶어서 표시
    (기술: IP 문자열 해시 → 고정 팔레트 매핑으로 결정적 색상 부여)
  - 빈 배경 드래그로 여러 행을 선택해 태그·분위기 클러스터를 일괄 변경 가능
    (기술: 마퀴 선택을 `.pcard` 전용에서 `.step2-card`까지 재사용 가능하도록 일반화, 선택 시 나타나는 일괄편집 툴바)

- **마퀴 선택 박스 개선**
  - 드래그 선택 영역이 그리드 컨테이너 밖으로 나가면 잘려 보이던 문제 수정
    (기술: 선택박스 엘리먼트를 컨테이너 자식 → `document.body` 자식 + `position:fixed`로 전환, 뷰포트 좌표 그대로 사용)

- **이전 작업 슬롯 캐시 (신규 기능)**
  - 탭을 닫거나 다른 탭으로 전환하면 그 시점 표 데이터(사진 썸네일 포함)가 최근 5개까지
    자동 저장, 1단계 화면에서 눌러서 바로 복원 가능 (insta-gen의 슬롯 캐시 패턴 이식)
    (기술: `localStorage` 기반 슬롯 배열, 사진/헤더는 축소 JPEG로 압축 저장, 용량 초과 시
    오래된 슬롯부터 사진 데이터를 비우고 텍스트만 남기는 폴백 포함 — 원본 소스 이미지는
    용량 문제로 캐시 대상에서 제외, 복원 시 표 정리(2단계)로 바로 이동)
  - 이 캐시로 복원하면 최종 내보내기 사진 화질이 원본보다 살짝 낮아짐(압축 저장이라)
    (기술: 정상 작업 경로가 아니라 사고로 탭을 닫았을 때의 안전망이라는 점을 화면
    안내문에 명시)

- **"다시 검출" 버그 수정**
  - 크롭을 이미 확정한 소스에서 "다시 검출"을 눌러도 재확정이 안 돼서 결과가 전혀 안
    바뀌는 것처럼 보이던 문제 수정
    (기술: `confirmSourceCrop()`의 `src.confirmed` 가드 때문에 확정 후엔 확인 버튼이
    계속 비활성 상태였음 → 확정된 소스에서 "다시 검출" 클릭 시 데이터 유실 경고 후
    `src.confirmed = false`로 되돌려 재확정 가능하게 함)
  - 재확정 시 이 소스가 예전에 만든 항목/사진/헤더를 정확히 지우고 새로 만들도록 개선
    (기술: 위치(`itemStartIndex`) 기준이 아니라 `src.itemIds` id 목록 기준으로 제거해서,
    2·3단계에서 순서를 바꾸거나 다른 항목을 지운 뒤에도 안전하게 동작)

- **클립보드 붙여넣기 지원 (신규 기능)**
  - 1단계에서 파일 선택 없이 캡처한 이미지를 Ctrl+V로 바로 붙여넣기 가능
    (기술: `document`에 `paste` 이벤트 리스너 추가, `clipboardData.items`에서 이미지
    항목을 찾아 기존 파일 업로드와 같은 `handleFilesSelected()` 경로로 처리)

- **"다시 검출" 클릭 피드백 부재 문제 수정**
  - 같은 이미지를 재검출하면 결과가 이전과 똑같을 수 있는데, 그 경우 화면이 전혀 안
    바뀌어서 버튼이 먹통인 것처럼 보이던 문제 수정
    (기술: 결과가 바뀌든 안 바뀌든 `.just-redetected` 클래스로 테두리를 잠깐 강조하고,
    버튼 문구도 "✓ 재검출 완료"로 1.4초간 바꿨다가 원복 — 실제로는 정상 동작하고
    있었으나 시각적 확인 수단이 없어서 오작동으로 오인되던 케이스)

- **1000px 아닌 폭 캡처에서 크롭 박스가 옆 칸을 침범하던 근본 원인 수정**
  - 칸 시작 위치(열/행 좌표)는 실제 픽셀을 스캔해서 정확히 찾고 있었지만, 정작 자를
    크기(카드 가로/세로 폭)는 "이미지 전체 폭 비율로 176px를 스케일링한 값"으로만
    추정하고 있었음 — 실제 캡처가 전체 폭에 정비례해서 커진 게 아니라 여백/줌 배율이
    제각각인 경우 이 추정치가 실제 카드 크기와 어긋나서, 옆 칸 사진 일부가 같이
    잘려 들어오거나 카드 일부가 잘려나가는 증상으로 나타남
    (기술: `detectRowTops`/`detectColStarts`가 이제 통과시킨 픽셀 런의 실제 길이도
    같이 반환하고, `detectGrid()`는 그 중 가장 많은 칸이 잡힌 스캔의 실측 길이 중앙값
    (`median()`)을 `cardW`/`cardH`로 사용 — 스케일 추정치는 실측값이 하나도 없을 때만
    쓰는 최후 대안으로 격하)
  - 합성 이미지(1000px/1404px/2160px 폭, 카드 크기가 전체 폭 비율과 다르게 설정된
    케이스)로 검증 — 기존 방식이면 카드 크기를 최대 27~180px까지 잘못 추정하던
    상황에서 실측값과 정확히 일치하는 결과 확인

- **텍스트 잠금(순서만 변경) 모드 추가 (신규 기능)**
  - 2단계에 "🔒 텍스트 잠금 (순서만 변경)" 체크박스 추가 — 켜면 2단계 표의 IP명/가격/
    태그/등급 입력·선택창이 전부 비활성화되고, 3단계에서도 항목 클릭 수정(✎)과 삭제
    체크박스가 비활성화됨. 카드 드래그로 순서를 바꾸는 기능은 잠금 여부와 무관하게
    항상 그대로 동작함
    (기술: `state.textLocked` 플래그 추가, `renderDataTable()`/`renderPreviewGrid()`에서
    `disabled` 속성으로 입력 요소만 막고 `card.draggable`은 그대로 유지, 3단계 진입 시
    `#textLockBanner`로 잠금 상태를 다시 안내)
  - 참고: 카드 드래그 재정렬(`moveItemsBeforeTarget`)은 원래부터 배열 순서만 바꿀 뿐
    `item.ip`/`price`/`tag` 등 텍스트 필드를 건드리지 않음 — 합성 데이터로 재정렬 전후
    텍스트를 대조해 확인함. 이 잠금 기능은 재정렬 자체의 버그를 고친 게 아니라, 재정렬
    도중 실수로 다른 칸을 잘못 눌러 값이 바뀌는 걸 막기 위한 안전장치임

- **3단계 "AI 분류로 정렬 보조" 기능 추가 (신규, 텍스트 전용 저비용 AI 호출)**
  - "자동 정렬 적용"이 등급표/클러스터에 없는 IP는 전부 '기타'로 동률 처리해서, 결과가
    이름 가나다순으로만 나오고 사전에 이미 그 순서였다면 눌러도 안 바뀐 것처럼 보이는
    문제가 있었음(버그 아님 — 구분할 사전 정보가 없어서 발생). 이를 보완하려고, 이미지를
    다시 보내지 않고 이미 확정된 IP명 텍스트만 보내 성향(genderLean)/분위기 클러스터만
    판단받는 새 API(`/api/classify-ip`)와 버튼을 3단계에 추가
    (기술: 신규 `classifyIpsForSort()` — 등급표 S/A에 있거나 이미 클러스터에 편입된 IP는
    후보에서 제외하고 중복도 제거해 최소 인원만 호출, 응답은 `subGrade`/`moodCluster`
    필드만 갱신하고 `ip`/`price`/`tag`는 절대 건드리지 않음, 완료 후 `autoSortItems()`
    자동 실행)
  - 1단계 "AI로 채우기"(이미지 인식)와 헷갈리지 않도록 두 버튼 모두에 서로를 안내하는
    문구를 추가하고, 3단계 버튼에는 호버 툴팁(`title`)도 붙임
  - 합성 데이터 + 모킹한 API 응답으로 검증: 등급표에 이미 있는 IP는 호출 후보에서 정확히
    제외되고, 응답에 따라 성향만 반영되며, 텍스트(ip/가격/태그)는 재정렬 전후로 완전히
    동일하게 유지되는 것까지 확인함

- **"AI로 분류할 IP가 없다" 안내 문구가 원인을 잘못 짚던 문제 수정**
  - 크롭 확정 직후(아직 IP명을 하나도 안 채운 상태)에 텍스트 잠금부터 켜면, IP명이
    전부 빈 칸이라 AI 분류 후보가 0개가 되는데 안내 문구는 무조건 "전부 이미
    등급표/클러스터에 등록돼 있다"고 떠서 실제 원인(잠금 때문에 IP명을 채울 수 없는
    상태)과 다르게 안내하던 문제 수정
    (기술: `classifyIpsForSort()`에서 빈 IP 개수를 별도로 세어, 후보가 0개일 때 "이미
    등록됨"과 "IP명이 비어있음(+잠금 중이면 해제 안내)"을 구분해서 안내)
  - 실제로 크롭 확정 직후 잠금부터 켠 뒤 3단계에서 버튼을 누르는 순서로 재현해서,
    새 안내 문구가 뜨는 것까지 확인함

- **1단계에 "🏷️ 라벨만 재인식" 기능 추가 (신규 — 재수입 전용, 저비용 OCR)**
  - "이미 만든 결과물을 다시 올려서 순서만 바꾸고 싶은데 IP명이 비어서 정렬을 못
    쓴다"는 문제의 근본 해결책. 사진 재인식(`AI로 채우기`)은 캐릭터를 사진으로
    알아맞혀야 하는 작업이라 비싸고, 이미 렌더링된 결과물에는 그럴 필요가 없음 —
    사진 아래 IP명/가격/태그 글자가 이미 깨끗하게 그려져 있으니 그 글자만 그대로
    옮겨 적으면 됨
    (기술: 신규 `cropLabelStripRegion()` — 사진(cardH) 영역은 아예 빼고 그 아래
    텍스트 라벨 줄만 잘라서 신규 API `/api/ocr-label`로 전송. 사전 매칭·성향 추측·
    분위기 클러스터 판단 없이 "보이는 글자 그대로 옮겨 적기"만 하는 훨씬 짧은
    프롬프트 사용. 합성 이미지로 측정한 결과 같은 행 기준 전송 이미지 높이가
    기존 방식 대비 약 63% 줄어듦)
  - 1단계에 AI 버튼 3개(AI로 채우기 / 라벨만 재인식 / 3단계의 AI 분류로 정렬 보조)의
    용도·비용을 한눈에 비교하는 표를 추가해 처음 쓰는 사람도 헷갈리지 않게 안내

- **3단계 카드 드래그를 네이티브 HTML5 드래그에서 마우스 이벤트 기반 커스텀 드래그로 전면 교체**
  - "드래그가 안 된다"는 신고를 실제 마우스 press-move-release 시뮬레이션으로 재현/검증한
    결과, 브라우저 네이티브 `draggable` API가 입력 방식에 따라 dragstart 자체가 안 잡히는
    경우가 있음을 확인(Playwright의 실제 드래그 시뮬레이션(`dragTo`)도 동일하게 재현 실패,
    반면 저수준 `mousedown`/`mousemove`/`mouseup` 이벤트로 직접 만든 드래그는 안정적으로
    동작함을 확인)
    (기술: `card.draggable`/`dragstart`/`dragover`/`drop` 제거, 신규 `startCardDrag()` —
    `mousedown` 시작 → `elementFromPoint`로 드롭 대상 카드 추적 → `mouseup` 시 
    `moveItemsBeforeTarget()` 호출. 이미 이 파일의 마퀴(배경 드래그 다중선택)에서 검증된
    것과 같은 패턴)
  - 항목이 많아(예 155개) 미리보기가 화면 여러 배 높이가 될 때, 네이티브 드래그가
    공짜로 제공하던 "뷰포트 가장자리에서 자동 스크롤" 기능이 사라지는 걸 막기 위해
    같은 동작을 직접 구현(뷰포트 위/아래 70px 이내에 마우스가 머물면 자동 스크롤)
  - **부수적으로 발견한 별도 버그**: 3단계(`previewGrid`)의 "빈 배경 드래그해서 여러 카드
    한번에 선택" 기능이 `attachMarqueeSelection()` 호출 시 두 번째 인자(카드 셀렉터)가
    빠져 있어서 처음부터 완전히 먹통이었음 — 셀렉터 인자를 채워서 수정
  - 합성 데이터로 실제 마우스 드래그(단일/그룹, 페이지 경계를 넘는 장거리 드래그+자동
    스크롤, 텍스트 잠금 중 편집 버튼 차단 유지)와 마퀴 다중선택까지 전부 재현 검증함
- **4단계 화면에 "여기는 순서 변경 화면이 아니다"라는 안내 추가**
  - 4단계엔 원래부터 카드 미리보기/드래그가 없었는데(3단계 전용 기능), 이걸 "드래그
    기능이 사라졌다"로 오인할 수 있어서 안내 문구를 추가해 명확히 함

- **"다시 검출"이 진짜로 의미 있게 동작하도록 수정 (근본 원인: 완전히 결정론적이었음)**
  - `detectGrid()`가 이미지 픽셀만 보는 순수 함수라 같은 이미지에 몇 번을 눌러도 항상
    똑같은 결과만 나올 수밖에 없었음(사용자가 "이 버튼이 대체 무슨 의미냐"고 지적) —
    이제 흰색 판정 임계값·사진 한 칸으로 인정하는 길이 범위를 다르게 하는 3가지 방식을
    순환하며 시도하도록 변경
    (기술: `detectRowTops`/`detectColStarts`가 `params` 인자로 임계값/범위를 받도록
    확장, `DETECT_VARIANTS` 3종 추가, `src.gridVariant`를 소스별로 유지하며 재검출 클릭마다
    `(variant+1) % 3`으로 순환. 배경이 살짝 회색빛(232,232,232)이라 기본값(흰색 임계값
    240)으로는 그리드를 아예 못 찾던 합성 이미지로 검증 — 1회 재검출 후 변형 2(임계값
    225)로 넘어가면서 정상 검출되는 것까지 확인함)
  - 재검출 버튼 문구도 "✓ 재검출 완료" → "✓ 다른 방식으로 재시도 (N/3)"으로 바꿔서
    실제로 다른 시도를 하고 있다는 걸 명시
- **헤더가 비어 보이는 문제 — 자동 검출값을 직접 조정할 수 있는 수동 보정 추가**
  - 헤더 크롭 높이(`rows[0]`)가 자동 검출값에만 의존해서, 이 값이 이미지마다 잘못
    잡히면 사용자가 고칠 방법이 전혀 없이 헤더가 비어 보이는 채로 남았음
    (기술: 소스 카드에 "헤더 높이(px) 직접 조정" 숫자 입력 추가, `src.headerHeightOverride`
    로 저장, 미리보기(파란 박스)와 실제 크롭(`confirmSourceCrop`) 양쪽에 반영. "다시 검출"
    시에는 새로 제안된 값으로 초기화됨)

- **슬롯 캐시: 완전히 동일한 데이터는 중복 저장하지 않도록 수정**
  - 탭을 여러 번 전환하면 그 사이 실제 작업이 없어도 매번 새 슬롯이 쌓여서, 같은 내용이
    시각만 다르게 여러 개 보이는 문제 수정
    (기술: `pushSlot()`에서 새 스냅샷과 가장 최근 슬롯을 `ts`(저장 시각) 제외하고
    비교(`snapshotContentKey`), 완전히 동일하면 새로 추가하지 않음)
  - 참고: "AI로 채우기"/"라벨만 재인식"으로 채운 IP명·가격·태그·성향(subGrade) 값도
    전부 `state.items`에 직접 저장되는 값이라, 슬롯 스냅샷에 그대로 포함되어 캐시에서
    복원해도 유지됨 — 실제로 확인함

- **"라벨만 재인식"이 지저분한 원본 상품명을 그대로 옮겨 적기만 해서 쓸모없던 문제 수정**
  - 처음엔 "이 도구가 이미 만든 결과물(이미 깨끗한 IP명만 있는 카드)을 재수입할 때 전용"
    으로 설계해서 "보이는 글자 그대로 옮겨 적기"만 시켰는데, 실제로는 원본 매장 스크린샷
    (상품명이 "[예약] GMG 콜렉션 17 지구연방군 세이라 마스 제복 버전ㅣ기동전사 건담"처럼
    지저분한 원문)에도 이 버튼을 썼고, 그 결과 IP명이 아니라 상품명 전체가 그대로 ip
    필드에 들어가버려 쓸모가 없었음
    (기술: `api/ocr-label.js`의 프롬프트/응답 스키마를 `api/parse-image.js`와 동일한 IP명
    추출 규칙(사전 매칭·"|" 구분자·상품 라인명 필터링·VTuber 규칙·태그 화이트리스트·
    분위기 클러스터·성향 추측)으로 교체 — 사진이 없어서 "글자가 흐리면 사진을 보고
    유추해라" 지시만 빼고 그 경우 uncertain으로 표시. `ocrLabelsForSource()`도
    `aiFillSource()`와 동일하게 사전/태그/클러스터 컨텍스트를 함께 보내고 결과의
    uncertain·moodCluster·genderLean까지 전부 반영하도록 수정)
  - 모킹한 API 응답으로 지저분한 원문 → 깨끗한 IP명 반영, 클러스터 자동 편입,
    등급표에 이미 있는 IP는 성향 추측을 건너뛰는 사전 우선 원칙까지 전부 확인함

- **3단계 미리보기: 같은 IP 색상 표시 + 같은 IP 기본 그룹 드래그 + 개별 이동 기능 추가**
  - 2단계 표에만 적용돼 있던 "같은 IP 카드 테두리 색 맞추기"를 3단계 미리보기 카드에도
    동일하게 적용 — 드래그 전에 어떤 카드끼리 같은 IP인지 바로 보임
    (기술: `renderPreviewGrid()`에도 `colorForIp(item.ip)`로 `card.style.borderLeftColor`
    지정, `.pcard`에 `border-left-width: 4px` 추가)
  - 카드를 드래그하면 같은 IP를 가진 카드 전부가 기본으로 함께 이동하도록 변경 (마퀴로
    직접 여러 장 선택했을 땐 그 선택이 우선)
    (기술: `startCardDrag()`의 `dragIds` 계산에 "마퀴 선택 없으면 `state.items`에서 같은
    `ip` 값을 가진 항목 전체를 자동 포함" 분기 추가)
  - 그룹 안에서 카드 하나만 예외적으로 개별 이동하고 싶을 때를 위해 두 가지 방법 추가:
    Alt(옵션)를 누른 채 드래그, 또는 카드 위 "1개만" 버튼을 눌러 다음 드래그 1회만
    개별 이동으로 만들기(무장 상태는 한 번 쓰면 자동 해제, 다시 눌러 취소도 가능) — 그룹에서
    영구히 빼는 기능은 필요 없다는 피드백에 따라 만들지 않음
    (기술: `singleMoveArmedId` 전역 상태(저장/캐시 대상 아님, 일회성) + `startCardDrag()`에
    `forceSingle` 인자 추가, IP가 1개뿐이거나 없는 카드엔 "1개만" 버튼 자체를 안 그림)
  - 합성 데이터로 같은 IP 카드 색상 일치, 평범한 드래그 시 그룹 전체 이동, Alt+드래그 시
    카드 1개만 이동, "1개만" 버튼 무장 후 평범한 드래그로도 1개만 이동하고 그 다음엔
    무장이 자동 해제되는 것까지 전부 재현 테스트로 확인함

- **배송비 표시 on/off 스위치 추가 (전체 항목 공통, 레이아웃 안 밀림)**
  - 3단계에 "🚚 배송비 표시" 체크박스 추가 — 꺼도 배송비 데이터(item.ship)는 그대로
    남고, 미리보기/최종 내보내기 이미지 둘 다에서 배송비 글자+장바구니 아이콘만 안
