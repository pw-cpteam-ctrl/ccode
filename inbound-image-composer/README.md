# 입고안내 이미지 자동 제작 툴

원본 입고안내 스크린샷(5열 그리드, 사진+구구절절한 상품명)을 업로드하면, 같은 사진을
재사용하되 텍스트만 "IP명 크게 + 라인업 태그 + 가격 작게 + 배송비"로 단순화한 4:5
(1080×1350) 이미지를 순서 지정해서 만들어 주는 셀프서비스 웹 도구다. 기획 배경은
핸드오프 문서(`design_handoff_inbound_image_tool/README.md`) 참고.

## 지금 상태

**기본은 배포/서버/API 키 없이 동작.** `index.html`을 더블클릭해서 브라우저로 열면 그대로
동작하는 바닐라 JS + Canvas 앱이다. 4단계 플로우를 화면 전환으로 분리해서 구현했다 —
중간 단계를 건너뛰고 바로 최종 렌더로 가지 않는다.

**"🤖 AI로 채우기"는 선택 기능**(1단계, 크롭 확정 후)이다. 대화형 AI로 하던 것과 똑같이
원본 스크린샷을 Claude Vision(`claude-sonnet-5`)에게 사진+텍스트를 함께 보내서 2단계 표(IP명/
가격/태그/배송비)를 자동으로 채운다. 이미지 1장당 대략 2~4센트 수준(100장 기준 2~4달러)이고,
결과는 항상 "⚠ AI 추정 - 확인 필요" 배지가 붙은 상태로만 표에 들어간다 — business-rules.md
원칙대로 AI가 자동으로 확정하지 않고 사람이 확인/수정해야 한다. 백엔드(`api/parse-image.js`)가
배포되어 있지 않으면 버튼을 눌러도 "정상입니다" 안내만 뜨고 수동 입력으로 진행하면 된다.

AI는 IP명/가격/배송비/태그뿐 아니라 **정렬에 쓰이는 분위기 클러스터, 등급표에 없는 IP의
성향(남성향/여성향)까지 함께 판단한다** — 기존 클러스터 사전(어느 클러스터에 어떤 IP들이
있는지)을 프롬프트에 같이 보내서, 확신 있게 읽은(uncertain=false) IP만 해당 클러스터에
자동으로 편입시킨다. 애매한 IP는 클러스터에 넣지 않고 사람이 "⚙ 사전 관리"에서 나중에
직접 배치하면 된다 — 잘못된 추정으로 사전을 오염시켜 정렬 기준이 계속 틀어지는 걸 막기
위함이다. 성향 추측은 **사전(등급표) 우선** 원칙을 따른다 — 이미 S/A급 사전에 있는 IP는
건드리지 않고, 사전에 없는 IP에 한해서만 AI가 B급 남성향/여성향으로 추측해 채운다(B/C
등급 자체는 판매 우선순위라는 영업 판단이라 항상 B로 채우고, 필요하면 사람이 직접 C로 조정).

1. **업로드 & 그리드 검출** — 원본 1~3장 업로드, 픽셀 스캔으로 그리드 좌표 검출(이미지
   가로폭이 1000px가 아니어도 비례 스케일링으로 정상 검출), 빨간 사각형(사진 크롭 범위)
   + 초록 점선(AI에게 실제 전송되는 텍스트 포함 범위) 디버그 오버레이로 시각 확인 후에만
   크롭 확정, 선택적으로 AI 자동 채우기
2. **표 정리** — 사진별 IP명/태그/가격/배송비 입력(AI가 채웠다면 검토/수정), IP명 사전
   자동완성 + "사전에 없음" 경고 (자동 확정 금지). 같은 IP명은 카드 왼쪽 테두리 색으로
   묶여 보이고, 빈 배경을 드래그하면 여러 행을 선택해 태그·분위기 클러스터를 일괄 변경 가능
3. **전체 미리보기 & 순서 정렬(분할 전, 1장)** — 카드를 드래그해서 순서 변경, 카드 클릭으로
   인라인 수정. 빈 배경을 드래그하면 여러 카드를 한 번에 선택해 묶어서 이동 가능(체크박스와는
   별개, 페이지 전체 어디로 드래그해도 선택 영역이 잘리지 않음). 체크박스로 선택 후 "선택
   삭제"를 누르면 확인 화면 없이 바로 삭제되고, "실행취소"로 가장 최근 삭제 1건을 되돌릴
   수 있다. 같은 화면에서 등급(S/A 사전 자동 판별 + B/C 성향별) → 스토어 성향 → 분위기
   클러스터 순으로 자동 정렬도 적용할 수 있고, 정렬 후에도 드래그(그룹 이동 포함)로
   미세조정 가능. "순서 확정"을 눌러야 다음 단계로 진행된다.
4. **최종 분할 & 내보내기** — 5×4(20개/페이지) 캔버스 렌더, JPEG quality 0.94 내보내기

## 폴더 구조

```
index.html            메인 UI (4단계 화면)
app.js                상태 관리 + 화면 로직
lib/grid-detect.js    reference/grid-detect.js 포팅 (알고리즘/수치 그대로)
lib/render-page.js    reference/render-page.js 포팅 (알고리즘/수치 그대로) + 미리보기용 연속 스트립 렌더 추가
lib/seed-data.js      business-rules.md의 IP명 사전/등급표/클러스터 초기값
lib/github.js         GitHub Contents API 커밋 헬퍼 (insta-gen과 동일 패턴)
api/load-dict.js      사전 데이터 조회 (Vercel 서버리스)
api/save-dict.js      사전 데이터 저장 (Vercel 서버리스, 명시적 저장 버튼에서만 호출)
```

## 데이터 모델

사진과 텍스트는 항상 분리 관리한다 (photoId로 연결) — 순서 재배치 시 재크롭이 필요 없다.

```js
photos: { [photoId]: HTMLCanvasElement }               // 원본 해상도 크롭
items: [{ id, photoId, ip, tag, price, ship, subGrade, pushToEnd }]  // 배열 순서 = 진열 순서
```

## 사전(계속 늘어나는 데이터) 관리 — 기본은 이 브라우저에만 저장

우측 상단 "⚙ 사전 관리"에서 IP명 사전 / 등급표(S·A급만) / 분위기 클러스터 / 스토어
프로필 / **상품 라인명**(넨도로이드·팝업퍼레이드처럼 IP명이 아닌 상품 라인 단어 목록 —
"AI로 채우기"가 이 단어들을 IP명으로 착각하지 않게 걸러줌, 새 상품 라인이 나오면 여기에
추가)을 편집할 수 있다. **"이 브라우저에 저장"** 버튼만 눌러도 `localStorage`에 즉시
저장되고 새로고침해도 유지된다 — 백엔드/배포/환경변수가 전혀 필요 없고 비용도 들지 않는다.

**"☁ GitHub에도 공유"는 완전히 선택 기능이다.** 여러 대의 기기·팀원끼리 사전을 맞추고
싶을 때만 아래 배포 단계를 거쳐 켜면 된다. 배포하지 않았다면 이 버튼을 눌러도 그냥
"백엔드가 없어서 이 브라우저 저장만 유지됩니다" 안내만 뜨고 끝난다 — 에러도 아니고
비용도 발생하지 않는다.

### (선택) 여러 팀원과 사전을 공유하고 싶다면 — Vercel 배포

1. Vercel 프로젝트 생성 시 Root Directory를 `inbound-image-composer`로 지정
2. GitHub에 `inbound-image-composer-data` 브랜치를 미리 만들어 둔다 (main과 분리 —
   그래야 사전을 저장할 때마다 Vercel이 재배포하지 않는다)
3. 아래 환경변수 등록

| 환경변수 | 용도 | 비고 |
|---|---|---|
| `GITHUB_TOKEN` | 사전 데이터 읽기/쓰기 | 이 레포에 `contents: write` 권한이 있는 fine-grained PAT |
| `GITHUB_OWNER`, `GITHUB_REPO` | 저장 대상 레포 | 예: `pw-cpteam-ctrl`, `ccode` |
| `GITHUB_BRANCH` | 사전 데이터 전용 브랜치 | 기본값 `inbound-image-composer-data` |

이 세 가지를 설정하지 않으면 `api/load-dict.js`, `api/save-dict.js`는 그냥 조용히
무시되고 이 브라우저 저장(localStorage)만으로 계속 정상 동작한다.

### (선택) "AI로 채우기"를 쓰고 싶다면 — Vercel 배포 + Claude API 키

이것도 배포 없이 쓸 수 있다 — 안 배포하면 그냥 "AI로 채우기" 버튼이 실패 안내만 띄우고
수동 입력으로 진행된다. 켜고 싶으면:

1. Vercel 프로젝트 생성 시 Root Directory를 `inbound-image-composer`로 지정 (사전 공유용
   배포를 이미 했다면 같은 프로젝트에 환경변수만 추가하면 됨)
2. [console.anthropic.com](https://console.anthropic.com)에서 API 키 발급 — **주의: 챗
   구독(Claude Pro/Max)과는 완전히 별개 결제 체계다. 챗 구독으로는 API를 쓸 수 없다.**
3. 환경변수 `ANTHROPIC_API_KEY` 등록

| 환경변수 | 용도 | 비고 |
|---|---|---|
| `ANTHROPIC_API_KEY` | `api/parse-image.js`의 이미지 인식 호출 | 결제 필요, `claude-sonnet-5` 모델 사용 |

이 환경변수가 없으면 `api/parse-image.js`는 500 에러를 돌려주고, 프론트는 그걸 "백엔드가
없어서 정상입니다" 안내로 바꿔서 보여준다 — 앱이 죽지 않는다.

## 의도적으로 뺀 것

- **OCR/AI 자동 인식은 항상 "선택 + 확인 필요" 상태로만 존재한다.** "AI로 채우기"가 채운
  값도 절대 자동 확정되지 않고 ⚠ 배지가 붙어 사람이 확인해야 확정된다 — business-rules.md의
  "IP명이 애매하면 반드시 확인받고 확정" 원칙을 그대로 따른 것.
- **B급 이하 등급은 여전히 사전에 상시 저장하지 않는다.** business-rules.md 원칙대로 S/A급만
  상시 사전으로 유지한다. 다만 사전에 없는 IP에 한해 AI가 성향(남성향/여성향)을 추측해
  등급 드롭다운을 미리 채워주긴 하지만, B/C 등급 자체(판매 우선순위)는 매번 사람이 확인/조정한다.
