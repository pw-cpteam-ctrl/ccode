# 입고안내 이미지 자동 제작 툴 — 기술 설계문서

## 프로젝트 목표

메가하우스 입고안내 스크린샷(5열 그리드)을 업로드하면, 텍스트만 단순화하고 사진은 재사용하는 4:5 카드(1080×1350)를 자동 생성해 주는 셀프서비스 도구. **백엔드 배포 없이 바닐라 JS + Canvas로 로컬 동작 가능.**

## 아키텍처

### 핵심 원칙

1. **로컬 우선** — 배포 없이 `index.html` 더블클릭으로 즉시 작동
2. **단계별 UI** — 4단계 플로우를 화면 분리로 실행 (중간 단계 스킵 불가)
3. **사진·텍스트 분리** — 순서 변경 시 재크롭 불필요
4. **AI는 선택 기능** — 백엔드 없으면 "정상입니다" 안내로 수동 진행
5. **사전은 누적** — localStorage 기본, 선택적으로 GitHub 동기화

### 데이터 흐름

```
Step 1: 업로드 & 그리드 검출
  ├─ 사진 업로드 (1~3장)
  ├─ grid-detect.js: 픽셀 스캔 → 5열 좌표 탐지 (이미지 실제 폭 기준 비례 스케일링,
  │   1000px 고정 가정 아님)
  ├─ 빨간 테두리(사진 크롭 범위) + 초록 점선(AI 텍스트 크롭 범위) 오버레이로 시각 확인
  └─ [선택] "🤖 AI로 채우기" → parse-image.js 호출

Step 2: 표 정리
  ├─ IP명 / 태그 / 가격 / 배송비 입력
  ├─ IP명 사전 자동완성 (seed-data.js + 누적)
  ├─ "사전에 없음" 경고 (자동 확정 금지)
  ├─ 같은 IP명끼리 카드 테두리 색으로 그룹 표시
  └─ 빈 배경 드래그로 여러 행 선택 → 태그/분위기 클러스터 일괄 변경

Step 3: 전체 미리보기 & 순서 정렬 (1장, 분할 전)
  ├─ 드래그로 순서 변경 (빈 배경 드래그 = 마퀴 선택으로 여러 카드 묶어서 함께 이동,
  │   선택박스는 body에 fixed로 붙어 있어 컨테이너 밖으로 나가도 안 잘림)
  ├─ 클릭으로 인라인 수정
  ├─ 체크박스 선택 → "선택 삭제" 클릭 시 확인 화면 없이 즉시 삭제, "실행취소"로 복구
  ├─ 등급 자동 판별(S/A: 사전, B/C: AI 성향 추측 + 수동 조정) → 스토어 성향 →
  │   분위기 클러스터 순 자동 정렬 (같은 화면에 통합 — 예전엔 별도 4단계였는데 두
  │   화면 모두 카드 드래그 이동을 지원해서 기능이 중복됐었음)
  └─ "순서 확정" 필수

Step 4: 최종 분할 & 내보내기
  ├─ render-page.js: 5×4 캔버스(20개/페이지) 렌더
  ├─ JPEG quality 0.94
  └─ ZIP 다운로드 (또는 이미지별 다운로드)
```

## 파일 구조 및 책임

| 파일 | 역할 | 상태 |
|------|------|------|
| **index.html** | 4단계 화면 UI, 상태 바인딩 | ✓ 완성 |
| **app.js** | 상태 관리 (Proxy 기반), 화면 전환 로직 | ✓ 완성 |
| **lib/grid-detect.js** | 픽셀 스캔 → 5열 그리드 좌표 탐지 | ✓ 포팅 완료 |
| **lib/render-page.js** | 카드 렌더(5×4), 미리보기 스트립 | ✓ 포팅 완료 |
| **lib/seed-data.js** | IP명 사전, 등급표, 클러스터, 스토어 프로필 초기값 | ✓ 완성 |
| **lib/github.js** | GitHub Contents API 커밋 (사전 저장용) | ✓ 완성 |

## 주요 기술 결정

### 1. 그리드 검출 알고리즘
- **방식**: 픽셀 단위 행 스캔 → 흰색 배경 비율로 분할선 탐지
- **정확도**: 참고 이미지 기준 5열 그리드 100% 정확 (검증 완료)
- **유지**: 참고 알고리즘(reference/grid-detect.js)과 동일 수치 사용

### 2. 카드 렌더링
- **포맷**: JPEG 1080×1350 (인스타그램 권장)
- **레이아웃**: 사진 큰 영역 + 텍스트 영역(IP명 크게, 가격/배송비 작게)
- **품질**: JPEG quality 0.94 (용량 vs 품질 트레이드오프)
- **다중 페이지**: 5×4(20개)씩 묶어서 렌더

### 3. 상태 관리
- **방식**: Proxy 기반 반응형 상태 (Vue/React 없이 구현)
- **저장소**: 메모리(기본) + localStorage(사진은 제외, 텍스트 데이터만)
- **범위**: IP명/태그/가격/배송비/등급/순서 누적 (여러 세션 유지)

### 4. AI 인식 (선택 기능)
- **모델**: `claude-sonnet-5` (haiku 대비 단가 2배지만, 사진+텍스트 복합 인식에서 정확도 개선)
- **호출 시점**: Step 1에서 "AI로 채우기" 버튼 클릭만 (자동 호출 없음)
- **결과**: 항상 "⚠ 추정 - 확인 필요" 배지로만 표시 (자동 확정 금지)
- **배치 방식**: 원본 그리드 1개 행(보통 5개)씩 나눠서 호출 (정확도 개선)
- **크롭 방식**: 항목별로 오려서 재조합하지 않고, 원본 레이아웃을 그대로 유지한 채 행
  단위로 통째로 자름 (재조합 과정에서 좌표가 어긋나 엉뚱한 항목이 뒤섞이는 문제 방지 +
  사진을 함께 보여줘 텍스트가 흐려도 캐릭터/작품을 유추할 수 있게 함)
- **스키마**: JSON Schema로 구조화된 응답 강제 (parsing 안정성)
- **상품명 파싱 규칙**: 상품명은 보통 "상품 라인명 + 캐릭터명 + | 작품명/소속그룹" 순서라,
  "팝업퍼레이드/넨도로이드/피그마" 같은 상품 라인명을 IP명으로 착각하는 문제가 있었다.
  "|" 뒤가 진짜 IP명이라는 규칙과 상품 라인명 블랙리스트를 프롬프트에 추가해 개선.
- **VTuber 규칙**: 홀로라이브/니지산지 등은 ip="VTuber" + tag=소속사명으로 분리해야 하는데,
  이 규칙이 프롬프트에 전혀 반영되어 있지 않아 소속사명이 IP명으로 잘못 나오던 문제를
  발견 → 프롬프트에 명시적으로 추가.
- **자동 분류 범위**: IP명/가격/배송비/태그(룩업·테노히라·메가캣)뿐 아니라 **정렬용 분위기
  클러스터까지 AI가 함께 판단**. 기존 클러스터 사전(`state.dict.moodClusters`)을
  프롬프트에 넘겨 "이 IP가 어느 클러스터와 어울리는지" 고르게 하고, `uncertain=false`인
  항목만 `app.js`의 `applyAiClusterResult()`가 실제로 사전에 편입시킨다 — 애매한 판단으로
  정렬 기준(클러스터 사전)을 오염시키지 않기 위함. IP명 사전처럼 이 편입도 누적되어
  다음 이미지 인식부터는 그 IP를 바로 기존 클러스터로 인식한다.
- **성향(genderLean) 추측**: 등급표(S/A급)에 없는 IP에 한해서만 AI가 남성향/여성향을
  추측해서 `subGrade`를 B급으로 미리 채운다(`applyAiGenderLean()`). 사전 우선 원칙 —
  이미 S/A급으로 등재된 IP는 절대 건드리지 않는다. B/C 등급 자체(판매 우선순위)는
  AI가 알 수 없는 영업 판단이라 항상 B로 채우고, 필요하면 사람이 드롭다운에서 C로 조정한다.

### 5. 사전 관리
- **기본 저장소**: localStorage (즉시 저장, 비용 무료)
- **선택적 동기화**: GitHub Contents API (여러 디바이스/팀원 공유)
- **병합 방식**: seed-data.js + GitHub + localStorage를 **합집합으로 병합**한다(예전엔
  localStorage에 값이 하나라도 있으면 시드 전체를 통째로 덮어써서, 나중에 시드에 추가된
  항목이 계속 안 보이는 버그가 있었다 — 예: 등급표에 새로 추가한 IP가 기존 사용자에게는
  적용 안 됨). 같은 키가 겹치면 로컬 편집이 우선하되, 시드 기본값 자체가 사라지지는 않는다.
- **등급 자동 판별**: S/A급은 사전에 있으면 자동, B/C급은 AI가 성향만 추측해서 미리
  채워주고(사전에 없는 IP에 한해), 최종 확정은 사람이 드롭다운에서 한다

### 6. 배포 전략
- **기본**: 배포 불필요 (바닐라 JS)
- **선택 1 (사전 공유)**: Vercel + GitHub Token (별도 브랜치 `inbound-image-composer-data`)
- **선택 2 (AI 인식)**: Vercel + Anthropic API Key
- **장점**: 배포 없는 선택지 = 비용 0원, 대기 시간 없음

## 트러블슈팅

### "AI로 채우기"가 동작하지 않음
→ `api/parse-image.js` 백엔드가 배포되지 않음. 배포하거나 수동 입력으로 진행.

### 사전 저장이 안 됨
→ GitHub 백엔드 미배포. localStorage는 정상 저장됨. 필요하면 배포 단계 실행.

### 그리드 검출이 틀림
→ 사진이 극단적으로 어둡거나 밝으면 실패할 수 있음. 분할선(흰 배경)이 명확한 사진 권장.

### 순서 정렬이 이상함
→ 등급/클러스터 사전이 비어 있을 수 있음. "⚙ 사전 관리"에서 확인/추가.
