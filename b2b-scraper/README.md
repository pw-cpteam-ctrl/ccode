# B2B 스크래퍼 (GoodSmile)

`product-sns-formatter`의 "① B2B에서 오늘 상품 가져오기" 버튼이 실제로 쓸 결과물
(`output.json` + 사진 파일들)을 만드는 도구. **팀원(비개발자)용 실행 방식**과
**개발자용 정찰(recon) 방식** 두 가지가 있음.

## ⚠️ 반드시 로컬(화면 있는) 컴퓨터에서 실행할 것

이 원격 개발 환경(화면 없는 샌드박스)에서는 실행할 수 없다 — 로그인 단계가 진짜 브라우저
창을 띄워서 사람이 직접 로그인해야 하고, 정찰/스크래핑 단계도 GoodSmile 쪽에서 이 환경의
IP를 막을 가능성이 있다. **아래 내용은 전부 각자 컴퓨터(Node.js 설치된 곳)에서 실행.**

---

## 팀원(비개발자)용 — 더블클릭 한 번

1. 이 폴더(`b2b-scraper`)를 통째로 받는다.
2. **Windows**: `run.bat`을 더블클릭 (원하면 바탕화면에 바로가기 만들어서 "굿스마일 상품
   가져오기"로 이름 붙여도 됨). **Mac**: `run.command`를 더블클릭 (최초 1회는 우클릭 →
   열기로 실행해야 보안 경고를 넘길 수 있음).
3. 검은 콘솔 창이 뜨고 진행상황이 한국어로 표시됨. 타이핑할 건 없음.
   - Node.js가 없으면 설치 안내가 뜸 → nodejs.org에서 LTS 설치 후 다시 더블클릭.
   - 최초 실행이면 "준비 중..." 문구와 함께 자동으로 라이브러리를 설치함 (1~2분).
4. 크롬 창이 뜨면:
   - **이미 로그인된 적 있으면** → 로그인 세션을 기억하고 있어서 아무것도 안 해도 자동으로
     넘어감.
   - **처음이거나 세션이 만료됐으면** → 그 창에서 GoodSmile 아이디/비밀번호로 로그인. 로그인
     완료되면 (엔터 칠 필요 없이) 자동으로 감지해서 이어서 진행됨.
5. 완료되면 콘솔에 "완료: N건 -> ..." 문구가 뜨고, `output/<날짜>/output.json`과
   `output/<날짜>/photos/` 폴더가 생성됨. 아무 키나 눌러 창을 닫으면 됨.
6. product-sns-formatter 페이지의 "🔄 B2B에서 오늘 상품 가져오기" 버튼을 누르고, 방금
   만들어진 `output.json` 파일과 `photos/` 폴더 안의 사진 파일들을 **함께 선택**하면
   (Ctrl/Cmd+클릭으로 다중 선택, 또는 폴더째 드래그) 상품 목록에 자동으로 채워진다.

**정직한 한계**: `.bat`/`.command`라 실행 중 검은 콘솔 창은 뜬다(진행상황 표시용). 다만
거기 뭘 입력할 필요는 없고, 실제 조작(로그인)은 전부 크롬 창에서 일어난다. 콘솔 창 자체를
없애려면 exe/GUI 포장이 필요한데, 이번 프로젝트 규모엔 오버킬로 판단해 `.bat`으로 마무리함.

## 개발자용 — 사이트 구조 정찰 (scrape.js 완성 전 필수)

`scrape.js`의 `TODO(recon)` 표시된 부분(목록→상세 URL 뽑는 법, 임베디드 JSON 안의 실제
필드 경로, 사진 URL 추출법, 로그인 URL 패턴)은 실제 GoodSmile 페이지 구조를 봐야 채울 수
있다.

```bash
cd b2b-scraper
npm install
node recon.js <오늘 상품 목록 페이지 URL>
```

`scrape.js`와 같은 전용 크롬 프로필(`chrome-profile/goodsmile`)을 공유하므로, `scrape.js`를
먼저 한 번 돌려서 로그인해뒀다면 `recon.js`는 로그인 없이 바로 정찰 가능. 처음이면 여기서도
크롬 창이 뜨고 로그인하면 자동으로 이어서 진행된다.

실행하면:
- `recon-output/`에 그 페이지의 HTML 전체와 스크린샷이 저장됨
- 터미널에 `__NEXT_DATA__` / `__PRELOADED_STATE__` / `__next_f` 마커가 있는지, 상품
  상세로 보이는 링크·이미지 URL 목록이 출력됨

**이 터미널 출력을 그대로 복사해서 알려주면**, 그걸 보고 `scrape.js`를 마저 완성할 수
있다. 상세 페이지 URL 하나도 같은 방식으로 한 번 더 정찰해두면 더 좋음:

```bash
node recon.js <상품 상세 페이지 URL 1개>
```

정찰이 끝나 `scrape.js`가 완성되면, 팀원은 위 "팀원용" 섹션대로 `run.bat`/`run.command`
더블클릭만 하면 된다 — 개발자가 직접 `node scrape.js`로 돌려도 동일하게 동작.

## 설계 메모

- **세션 기억 방식**: 로그인 스크립트를 따로 두지 않고 `scrape.js`(및 `recon.js`) 자체가
  전용 크롬 프로필(`chrome-profile/goodsmile`)로 세션 유무를 확인 → 없거나 만료되면 로그인
  창을 띄우고 URL 변화로 로그인 완료를 자동 감지 → 있으면 곧바로 스크래핑, 한 흐름으로 처리.
  (기존에 쓰던 "터미널에서 엔터 치면 storageState 저장" 방식은 비개발자에게 낯설어서 폐기.)
- **가벼운 설치**: `playwright` 대신 `playwright-core`를 의존성으로 사용 — 팀원 PC에 이미
  깔린 진짜 크롬(`channel: 'chrome'`)을 그대로 쓰므로 브라우저 ~300MB 자동 다운로드가 없고,
  최초 `npm install`이 수 MB 수준으로 가벼움.
- 봇탐지 위장(`applyStealth` 등)은 옮기지 않았음 — B2B 업무포털은 인스타/X처럼 엄격하지
  않을 가능성이 높아서. 로그인이 자꾸 자동화로 의심돼 막히면 그때 강화하면 됨.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `run.bat` / `run.command` | 팀원용 더블클릭 런처 (Windows / Mac) |
| `scrape.js` | 메인 스크립트 — 세션 확인/로그인 대기 → 스크래핑까지 한 번에. `TODO(recon)` 부분은 정찰 결과 필요 |
| `browser-stealth.js` | 전용 크롬 프로필 세션 열기 + 자동 로그인 감지 도우미 |
| `recon.js` | 페이지 구조 정찰 (개발자용) |
| `extract-helpers.js` | 페이지에 박힌 JSON 추출 헬퍼 (sns-report-scraper 재사용) |
| `download-image.js` | 로그인 세션으로 사진 다운로드 |
| `format-output.js` | 필드를 output.json 형태로 조립 |
