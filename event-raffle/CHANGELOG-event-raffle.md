## 2026-07-18 (V1.1 ~ V1.11 하이브리드 필터링 재설계 + UX 대개선)

- **필터링 백엔드 전환**
  - AI 필터링 호출 대상을 Gemini에서 Claude로 전환
    (기술: `api/filter.js`가 `@anthropic-ai/sdk`로 `claude-opus-4-8` 호출, `output_config.format`(JSON 스키마)으로 응답 형식 강제, 환경변수 `GEMINI_API_KEY` → `ANTHROPIC_API_KEY`로 문서 전체 정정)
  - 실제 필터링에 쓰는 모델을 더 가벼운 모델로 교체
    (기술: `claude-opus-4-8` → `claude-sonnet-5`, `output_config.effort: 'medium'` 명시)

- **비용 절감 재설계 (신규 기능)**
  - 조건을 "코드로 판단 가능한 규칙"과 "AI 판단이 필요한 나머지"로 나눠서, 코드 규칙은 브라우저가 참가자 전원에게 무료로 적용하고 AI에게는 남은 소수만 보내는 방식으로 변경
    (기술: `api/plan-filter.js` 신규 — 참가자 데이터는 보내지 않고 조건 텍스트만 분석해 `codeRule`/`subjectiveCondition` 반환, 코드 규칙이 없거나 실행 오류 시 기존 방식(전체를 AI로)으로 자동 폴백)
  - 참가자 수가 늘어나도 AI에게 보내는 데이터량이 늘지 않도록 구조 변경
    (기술: 코드 규칙 적용은 클라이언트 루프, AI 호출은 narrowed 후보 집합에만 수행 — 2천 명 규모에서도 비용·타임아웃 위험 감소)
  - 필터링 실행 중 중복 클릭 방지 + AI 판단 대상이 200명을 넘으면 예상 비용을 보여주고 확인받은 뒤 진행
    (기술: 버튼 요청 중 비활성화, `ROW_CONFIRM_THRESHOLD=200` 초과 시 `confirm()` 다이얼로그)
  - "정성스럽게/진심이 담겼는지" 같은 정성 조건을 AI 판단 대신 글자 수 기준으로 코드 처리
    (기술: plan-filter 프롬프트에 `row["본문"].length >= N` 형태 codeRule을 기본값으로 생성하도록 지침 추가 — 완벽한 정확도보다 비용 절감 우선 트레이드오프 명시)
  - 파일 업로드 시 본문 글자 수 분포를 계산해서 보여주고, 그 기준으로 직접 조건을 거는 "글자 수 조건" 칸 추가
    (기술: 구간을 고정값이 아니라 이번 파일의 최대 글자 수에 맞춰 5구간으로 자동 분할, 최소/중앙값/평균/최대 계산, 기본값은 중앙값으로 프리필 — 이 조건은 100% 코드 처리라 AI 호출 없음, 기존 자유 텍스트 기반 AI 추측 폴백은 그대로 유지)

- **안정성 개선**
  - 코드 규칙 적용 후 후보가 0명이 되면 빈 데이터가 그대로 AI 판단 API로 넘어가 서버 에러가 나던 문제 수정
    (기술: `preCandidateRows.length === 0` 분기 추가 — AI 호출 없이 바로 "0명 선정됨" + 적용된 규칙을 표시)
  - 네트워크 실패와 서버 오류를 구분하지 않고 항상 "API 키 미설정"으로 안내하던 에러 메시지 수정
    (기술: `catch` 블록에서 `TypeError`(네트워크)와 그 외 서버 응답 오류를 분리 처리)
  - 스크래퍼 원본 컬럼명(`css-1jxf684`, `__EMPTY` 등 뒤섞인 이름)이 들어와도 안전하게 처리되도록 plan-filter 프롬프트 보강
    (기술: 컬럼명이 의미 없는 문자열일 수 있다는 점, 확신 없으면 codeRule을 무리하게 만들지 말고 AI 판단으로 넘기라는 지침 추가, 예시 데이터 3개 → 8개로 확대)
  - BOM 없는 한글 CSV 업로드 시 인코딩이 깨지던 문제 수정
    (기술: `.csv` 파일은 `file.text()`로 읽어 UTF-8로 명시 디코딩)

- **조건 입력 UI 개선**
  - 조건 입력을 텍스트 한 칸에서 여러 칸(조건1/조건2/...) + 그리고(AND)/또는(OR) 선택으로 변경
    (기술: 왼쪽부터 순서대로 묶어 `(A) AND (B) OR (C)` 형태 문자열로 조합해 서버에 전달 — 자연어 문장의 연산자 우선순위 모호함 제거)
  - 업로드 직후 "댓글/후기 본문" 컬럼과 닉네임 컬럼을 사용자가 직접 지정하도록 변경
    (기술: 평균 문자열 길이가 가장 긴 컬럼을 본문으로, 숫자가 아닌 첫 컬럼을 닉네임으로 자동 추정해 미리 선택, 필터링/AI 판단은 오직 이 컬럼만 사용해 무관한 컬럼 전송 자체를 차단)

- **표/결과 화면 개선**
  - 표 셀이 길게 잘려서 전체 내용을 볼 방법이 없던 문제 수정
    (기술: 행 클릭 시 그 행만 줄바꿈 허용/너비 제한 해제(`collapsed` 클래스 토글), 기본은 펼침 상태로 전환하고 "전체 접기/펼치기" 버튼 추가)
  - 당첨자 카드에 원본 데이터가 안 보이던 문제 수정
    (기술: `<details>`/`<summary>` "원본 데이터 보기" 토글 추가, 클릭 시 전체 컬럼 값 표시)
  - 결과 다운로드를 JSON에서 엑셀로 변경
    (기술: `XLSX.utils.json_to_sheet` + `XLSX.writeFile`로 브라우저 자체 xlsx 다운로드)
  - 추첨에 실제로 쓰인 난수 값을 화면·다운로드·저장 기록에 표시
    (기술: 추첨마다 `{random, poolSizeBefore, idx}` 기록 — "코드가 진짜 무작위로 뽑았다"는 근거를 눈으로 확인 가능)
  - 참가자와 무관한 랭킹 집계 컬럼(`__EMPTY_1~9` 등) 때문에 본문 컬럼이 좁게 눌리던 문제 수정
    (기술: 값이 채워진 비율이 10% 이하인 컬럼은 미리보기/후보 표에서 제외(`getMeaningfulColumns`, 실측 기준 1%→10%로 조정), 필터 이후 후보 표는 후보 집합 기준으로 다시 계산, 본문 컬럼 헤더에 `min-width` 지정)

- **저장 기능 개선**
  - 당첨자 명단을 GitHub 레포에 자동 커밋하는 기능 추가
    (기술: `api/save-result.js` 신규 — `GITHUB_TOKEN` 환경변수로 Contents API 호출, 참가자 원본 데이터·댓글은 포함하지 않음)
  - 회차마다 별도 파일을 만들던 방식을 누적 기록 파일 하나로 변경
    (기술: `event-raffle/results/history.json`에 계속 append, 동시 저장 충돌(sha 낡음) 시 최신 내용을 다시 받아 병합 후 최대 3회 재시도)

- **데이터 캐싱 (신규 기능)**
  - 참가자 파일을 이 PC 브라우저에만 임시로 저장해, 다음에 다시 열어도 자동으로 이어서 쓸 수 있게 함
    (기술: `localStorage` 기반 슬롯 캐시, 최근 5개까지 보관(`SLOT_MAX`), 내용 중복 시 저장 생략, 저장 공간 부족 시 오래된 슬롯부터 무거운 필드 제거 후 재시도, 슬롯별 삭제 버튼 제공, 업로드/컬럼 선택 변경/탭 전환·닫기 시점에 자동 저장)

- **기타**
  - 링크로 바로 들어온 사람도 도구를 이해할 수 있도록 소개 카드 추가
    (기술: 제목 아래 번호 목록 카드, 참가자 데이터 저장 범위를 정확히 안내하는 문구 포함)
