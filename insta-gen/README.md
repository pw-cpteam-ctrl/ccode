# 인스타 카드 생성기

사진 **1장**을 올리면 인스타용 카드(1080×1350)를 만들어 주는 웹 도구입니다.
빌드/서버 없이 브라우저로 바로 엽니다.

> **파일 2개를 같은 폴더에 두고 배포하세요: `index.html` + `dict.js`**
> `dict.js` = 캐릭터·작품·라인 사전(단일 소스). 이 파일만 고치면 인식 대상이 늘어납니다.
> 둘이 분리되면 인식이 안 되니, 팀에 보낼 때 반드시 같은 폴더에 함께 두세요.

## 레이어 구조 (3층)

```
1층  텍스트     ← editable 필드에 붙여넣고 수정
2층  사진       ← 업로드한 사진, 웹에서 크롭(드래그/줌)만 가능
3층  배경지     ← 같은 원본 이미지를 불투명 70% + 흐림으로 깔기
```

로고·한자·배지·누끼 **없음**. 폰트는 **Pretendard (300~800)** 로 통일.

## 텍스트는 어디서 오나 — 버튼 2개

`2 · 원문 입력`에 일본어 원문을 그대로 붙여넣고 아래 두 버튼 중 하나를 누르면
헤드라인·작품·라인·캐릭터·예약일을 자동 인식해 `3 · 텍스트` 필드를 채운다.

1. **[무료로 채우기]** — 도구 안에 내장된 **일본어→한글 사전 + 템플릿 파서**로 인식(AI
   호출 없음 = 과금 없음, 오프라인에서도 동작). 사전에 없는 신규 작품/캐릭터만
   `✓ … 못 채운 항목` 안내가 뜨고, 그 부분만 3번에서 직접 입력.
2. **[AI로 채우기]** — Claude API로 원문을 분석. 사전에 없는 새 작품/캐릭터도 인식할
   확률이 높지만, 누를 때마다 API 토큰 비용이 든다.

사전은 `dict.js`(works/chars/versions + HEADS/LINES + KR_WORKS/KR_ALIAS) **한 곳**이다.
**새 표기가 확정되면 `dict.js`만 갱신**한다.

### 추가 동작
- **이전 입력 5칸 캐시**: 원문을 채울 때마다 직전 작업이 슬롯에 남는다(브라우저
  localStorage 저장, 사진 제외). 슬롯을 누르면 그 내용으로 되돌린다.
- **불렛(•) 기본 ON**: 카드·원고 모두 캐릭터명 앞에 `• `가 붙는다.
- **작품명 더미**: 비면 `[작품명]`이 강제로 남아 수정을 유도한다.
- **테노히라 등 접두 라인**: 타이틀은 `G.E.M. 시리즈`, 제품명 앞에 `테노히라`를 붙인다.
- **호칭 변환**: `くん→군`, `さん→상`, `ちゃん→짱`, `さま/様→님`.

## 쓸수록 똑똑해지는 무료 사전 (자동 학습)

[AI로 채우기]를 쓸 때마다 결과가 버려지지 않고 저장소에 계속 쌓인다. 같은 작품/캐릭터의
(일본어→한글) 짝이 **2번째로 등장하면 자동으로 `learned-dict.json`에 반영**되고,
그 다음부터는 [무료로 채우기]도 그 작품/캐릭터를 알아본다. `dict.js`(사람이 직접 관리하는
사전)는 건드리지 않고, 별도 파일에서 자동으로 채워지는 구조라 안전하다.

- 원문 로그: `insta-gen/logs/YYYY-MM-DD.jsonl`
- 등장 횟수 집계: `insta-gen/learned-counts.json`
- 2회 이상 등장해 실제로 반영된 사전: `insta-gen/learned-dict.json`

이 데이터들은 **`main`이 아니라 `insta-gen-data`라는 별도 브랜치**에 커밋된다. `main`에
커밋하면 Vercel이 그때마다(AI로 채우기를 쓸 때마다) 사이트를 재배포해서 빌드 시간을
낭비하기 때문 — `insta-gen-data`는 Vercel 배포 대상이 아니라 아무리 커밋해도 재배포를
안 일으킨다. `api/learned-dict.js`가 이 브랜치의 `learned-dict.json`을 대신 읽어서
index.html에 돌려주고, index.html은 그걸 fetch해서 무료 사전에 병합한다.

이 기능을 켜려면 Vercel의 insta-gen 프로젝트에 아래 환경변수를 추가해야 한다
(`ANTHROPIC_API_KEY`와 별개). 없으면 학습 저장만 조용히 스킵되고, AI 파싱 자체는 그대로 동작한다.

| 환경변수 | 값 |
|---|---|
| `ANTHROPIC_API_KEY` | AI 파싱용 (필수) |
| `GITHUB_TOKEN` | 이 저장소에 `contents: write` 권한이 있는 PAT |
| `GITHUB_OWNER` | 저장소 소유자 (예: `pw-cpteam-ctrl`) |
| `GITHUB_REPO` | 저장소 이름 (예: `ccode`) |
| `GITHUB_BRANCH` | **`insta-gen-data`** (데이터 전용 브랜치, `main` 아님) |

## 사용법

1. 제품 사진을 Claude에게 줘서 텍스트 4줄(헤드라인/타이틀/제품명/푸터)을 받는다.
2. `index.html` 을 브라우저로 연다 (더블클릭).
3. 같은 사진 1장 업로드 → 배경지(3층)와 사진(2층)에 자동 적용.
4. 받은 텍스트를 필드에 붙여넣고 editable로 2차 수정.
5. 사진을 **드래그해서 위치 조정**, 휠/슬라이더로 **확대**(크롭).
6. 배경 불투명·흐림 슬라이더로 미세 조정.
7. **PNG 다운로드** 클릭 → `insta-card.png` 저장.

## 커스터마이즈 포인트 (index.html 내부)

- `FRAME` — 사진 프레임 위치/크기/모서리
- `render()` 의 `fillText` 좌표·`setFont(weight,size)` — 텍스트 배치/굵기
- 상/하 스크림 그라데이션 — 텍스트 가독성
