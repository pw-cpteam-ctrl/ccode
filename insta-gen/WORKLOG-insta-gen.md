## 2026-07-15

### 자동 학습 파이프라인이 실제로 잘 작동하는지 재확인
- **무슨 문제/요청이 있었나**: "AI로 채우기를 쓸수록 무료 사전이 알아서 똑똑해지는 기능이 지금 정상 작동하는 게 맞냐"는 질문
- **확인해보니**: 라이브 사이트에 실제로 테스트 문구를 두 번 보내서 끝까지 직접 확인 — 첫 호출에서 일본어 원문 그대로 표기(`workJp`/`productJp`)가 잘 나오고, 같은 짝이 두 번째로 나오는 순간 학습 데이터가 배포와 분리된 `insta-gen-data` 브랜치에 정확히 쌓이며, `main`은 전혀 건드리지 않음을 확인함
- **어떻게 처리했나**: 테스트로 만든 가짜 데이터(`파이프라인 검증용 테스트` 등)는 실제 사전을 오염시키지 않도록 바로 삭제
- **결과**: 정상 작동 확인됨 — 테스트 데이터도 깨끗이 정리됨

### 허브 페이지로 들어오면 AI로 채우기가 항상 실패하는 문제
- **무슨 문제/요청이 있었나**: "AI로 채우기 실패: 서버 오류 (405)"가 뜨고 전혀 도움을 못 받는다는 신고
- **확인해보니**: 실제 배포 주소(`ccode-omega.vercel.app`)에 직접 접속하면 정상 동작했지만, 사용자는 허브 페이지(`pw-cpteam-ctrl.github.io/ccode/`)를 거쳐 들어오고 있었음. 허브는 정적 파일만 서빙하는 GitHub Pages라 서버 기능(`/api/parse`)이 아예 없고, 이 경로로 들어온 페이지가 그 주소로 요청을 보내면 GitHub Pages가 "POST 지원 안 함(405)"으로 응답하던 것이 원인 — 즉 코드 버그가 아니라 잘못된 경로로 들어와서 생긴 문제
- **어떻게 처리했나**: 허브 페이지의 "인스타 카드 생성기" 카드 링크를 상대경로(`insta-gen/index.html`)에서 실제 Vercel 배포 절대주소(`https://ccode-omega.vercel.app/`)로 변경. 다른 카드들도 서버 기능이 필요한지 점검했고, 유일하게 필요했던 "상품 SNS 포맷터" 카드는 이미 절대주소였음을 확인
- **결과**: `main`에 반영 후 GitHub Pages 재빌드까지 확인, 허브를 거쳐 들어와도 AI로 채우기가 정상 작동함

### 슬롯+캐싱 기능 코드를 다른 프로젝트에서 재사용하고 싶다는 요청
- **무슨 문제/요청이 있었나**: insta-gen에 있는 "이전 입력 5칸 캐시" 기능을 다른 프로젝트에도 붙이고 싶다며 관련 코드만 간단히 공유해달라는 요청
- **어떻게 처리했나**: localStorage 기반 슬롯 저장/복원/렌더링 코드(HTML+CSS+핵심 함수)를 정리해서 채팅으로 전달. 프로젝트마다 다른 부분(현재 화면 상태를 스냅샷으로 만드는 부분, 슬롯 클릭 시 복원하는 부분)은 별표로 표시해 그 프로젝트 사정에 맞게 고치도록 안내
- **결과**: 코드만 전달, insta-gen 자체는 변경 없음

### 화면 레이아웃 전면 개편 (여러 차례 목업으로 검토 후 반영)
- **무슨 문제/요청이 있었나**: 팀원이 실제로 써보니 각 칸이 무슨 역할인지 감을 못 잡고, 화면 배치도 답답하고 비효율적이라는 지적. 요청 사항이 많고 복잡해서 "코드로 바로 만들지 말고 그림으로 미리 보여달라"는 요구가 있었음
- **어떻게 처리했나**: 실제 코드/스타일 그대로 복사한 임시 목업 파일을 만들어 Playwright로 스크린샷을 찍어가며 여러 차례 검토받음. 중간에 손그림 주석을 오독해 엉뚱하게 고친 적이 여러 번 있었고("2열 분리"로 잘못 이해했다가 "박스 자체를 분리하라"는 뜻이었음이 뒤늦게 밝혀짐 등), 그때마다 사용자가 다시 스샷에 표시해준 내용을 재확인하며 바로잡음. 최종 확정된 내용:
  - 카드 미리보기를 75%로 축소하고 그 옆에 이전 입력 슬롯 5칸을 항상 표시 (빈 슬롯도 자리를 차지해서 기능이 있다는 걸 안 써봐도 알 수 있게)
  - "텍스트 입력" 칸을 미리보기 바로 밑으로, "원고(캡션)" 칸을 오른쪽으로 서로 위치를 바꿈
  - "사진 올리기"와 "원문 입력"을 한 박스 안에 가로로 나란히 배치해서 화면 높이를 줄임 (사진 칸 높이는 버튼 줄 끝나는 지점에 정확히 맞춤)
  - "두 버튼이 뭐가 다른지" 설명은 별도의 독립된 박스로 분리 (원래는 흰 박스 안에 있었는데 안 쓴 걸 잘못 알아듣고 처음엔 2열로 쪼갰다가 다시 되돌리고 박스 자체를 분리함)
  - 각 칸 제목 밑에 무슨 역할인지 알려주는 설명 문구를 크게(16px, 굵게) 추가
  - 헤드라인·작품명 입력칸을 가로 2열로 배치
  - 진행 중 슬라이더가 카드 밖으로 튀어나가는 버그, 미리보기 밑 안내문구가 캔버스랑 안 맞고 한쪽으로 쏠리는 버그를 추가로 발견해 같이 수정
- **결과**: 실제 파일에 반영 후 로컬에서 Playwright로 슬롯 채우기·복원·삭제 기능이 다 정상 작동하는지, PNG 다운로드용 실제 해상도(1080×1350)가 미리보기 축소와 무관하게 그대로인지 직접 확인. `main`에 커밋/배포하고 라이브 사이트에서도 반영된 것 확인함
