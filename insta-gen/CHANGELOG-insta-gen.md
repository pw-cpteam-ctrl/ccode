## 2026-06-22 (V1.0 최초 프로토타입)
- **핵심 기능**
  - G.E.M. 시리즈 인스타 카드 자동 생성기 프로토타입 최초 추가 → 단일 HTML 파일 도구로 전환
  - 브라우저 내 OCR 제거 — 텍스트는 Claude 이미지 인식 결과를 받아 쓰는 방식으로 전환
  - 사진 프레임 위치 조정, 제품명 2줄+불렛 토글, 클립보드 붙여넣기, 배경 블러 강화
  - 여백·폰트 크기 스테퍼 배치 조정

- **파싱 방식 실험**
  - 사진에서 IP/캐릭터를 Opus로 자동 인식하는 기능 추가 → 이후 API 비용 없는 원문 붙여넣기 방식으로 교체
  - 헤드라인/작품/라인 드롭다운 도입, 원고(캡션) 생성기 최초 추가
  - 캐릭터 사전 분리, 문맥 기반 캡션 푸터·예약판매일 로직 추가
  - 적용 버튼(원문 블록 → 필드 자동채움), 원문 필수 입력, 카드 색상 구분(노란/파란 구역) 추가
  - 일본어 원문을 내장 사전으로 파싱하는 기능 최초 추가
  - 가타카나→한글 음역 폴백, 미등록 한자는 kuromoji.js로 지연 로드해 처리

## 2026-06-23 (V1.1 사전 안정화 및 원고 개편)
- **사전/파서 개선**
  - 작품 등록 확대(마도카☆마기카 등), 요일 파싱, 해시태그 문자 정리
  - 한국어 자유입력 파싱 안정화 (오퍼시트 기반)
  - 사전을 `dict.js` 하나로 통합, 중복 제거
  - 일본어/한국어 자유입력 모두 지원, `／` 구분자·약칭·대소문자 처리 추가
  - 헤드라인/라인 전체 문구를 캐릭터 줄 strip 대상에 추가

- **원고 및 UX**
  - 테노히라 접두어, 호칭 변환(`くん→군`, `さん→상`, `ちゃん→짱`, `様→님`) 추가
  - 이전 입력 5칸 캐시, 불렛(`•`) 기본 ON, 작품명 더미값(`[작품명]`) 추가
  - 원고 양식을 "긴 작품/짧은 작품"으로 개편, 사진 캐시, 트위터 글자수 카운터 추가
  - 글자수 카운터를 트위터 가중치(한글=2, 280자 한도)로 수정
  - 사진 위치 드래그 복구, 이전입력 박스를 원고 칸으로 이동, 띄어쓰기 검사 추가

## 2026-07-08 (V1.6 타이틀 자동 줄바꿈 및 자동 학습 파이프라인)
- **버그 수정**
  - `esc()` 정규식 이스케이프 오류 수정 (수동 파일 재구성 과정에서 발생한 회귀)

- **카드 디자인 개선**
  - 작품명+시리즈명 조합이 길어서 카드 타이틀이 답답해 보이던 문제 → 실측 폭 기반으로 자동 두 줄 줄바꿈 (짧으면 한 줄 유지), 제품명 블록 위치도 함께 밀어서 겹침 방지

- **UI 개선**
  - "적용 → 3번 자동 채우기" / "✦ AI로 파싱" 버튼을 "무료로 채우기" / "✦ AI로 채우기"로 개명
  - 두 버튼의 차이(오프라인 가능 여부, 토큰 비용 누적 가능성)를 쉬운 말로 설명하는 안내 박스 추가

- **자동 학습 파이프라인 (신규 기능)**
  - AI로 채우기 결과의 (일본어→한글) 작품명/캐릭터명 짝을 GitHub 저장소에 계속 기록, 같은 짝이 2번째로 등장하면 무료 사전(`learned-dict.json`)에 자동 반영하는 파이프라인 추가 — 쓸수록 무료 채우기의 인식률이 올라가는 구조
  - 학습 데이터를 배포 대상인 `main`과 분리된 `insta-gen-data` 브랜치에 저장하도록 변경 — AI로 채우기를 쓸 때마다 불필요한 재배포가 발생하던 문제 방지

## 2026-07-15 (V1.8 레이아웃 전면 개편 및 배포 경로 수정)
- **버그 수정**
  - 허브 페이지를 거쳐 들어오면 AI로 채우기가 항상 오류로 실패하던 문제 수정
    (기술: 허브의 카드 링크가 상대경로라 GitHub Pages 도메인으로 이동됨 → 정적 호스팅엔 서버리스 함수가 없어 `/api/parse` 호출이 405로 실패 → 허브 카드 링크를 Vercel 배포 절대주소로 변경)
  - 슬라이더가 좁은 칸에서 카드 밖으로 튀어나가던 문제 수정
    (기술: flex 아이템의 기본 `min-width:auto`로 인해 `<input type=range>`가 안 줄어들던 버그, `min-width:0` 추가)

- **레이아웃 전면 개편**
  - 카드 미리보기를 75% 축소하고 옆에 이전 입력 슬롯 5칸을 항상 표시(빈 슬롯도 자리 유지)해 여러 상품을 빠르게 오가며 작업 가능
    (기술: `#stage`는 CSS 표시 크기만 축소, canvas width/height 속성(1080×1350)은 그대로 둬 다운로드 화질 무관 — `.slotcol`/`.slotsq` 구조로 `renderSlots()` 재작성)
  - "텍스트 입력" 영역을 미리보기 바로 아래로, "원고(캡션)" 영역을 오른쪽 패널로 위치 교체
  - "사진 올리기"와 "원문 입력"을 한 박스 안에 가로 2열로 압축해 화면 높이 절약
    (기술: `.combo12`/`.combo-photo`/`.combo-text` flex 레이아웃, 사진 칸 높이를 버튼 줄 하단에 맞춤)
  - "두 버튼이 뭐가 다른지" 설명을 별도의 독립된 박스로 분리
  - 각 섹션 제목 아래 무엇을 하는 칸인지 설명하는 문구 추가
  - 헤드라인·작품명 입력칸을 가로 2열로 배치해 세로 공간 절약
