# CHANGELOG.md 작성 규칙

메가하우스 마토메 프로젝트의 CHANGELOG를 기반으로 한 통일된 작성 규칙입니다.

---

## 📋 기본 포맷

```markdown
## YYYY-MM-DD (VX.X 버전 제목)
- **섹션명**
  - 상세 항목 1
  - 상세 항목 2
  
- **다른 섹션명**
  - 상세 항목
```

---

## 📐 구조 규칙

### 1. **버전 헤더 (첫 줄)**
```markdown
## YYYY-MM-DD (VX.X 제목)
```
- 날짜: ISO 형식 (2026-07-03)
- 버전: `V1.0`, `V1.4` 등 (초기 버전은 V1.0, 첫 업데이트는 V1.1)
- 제목: 해당 버전의 주요 테마 (예: "2페이지 지원", "팀 공유 기능")
  - 한 문장 또는 짧은 구문
  - 마침표 없음

### 2. **섹션 (중제목)**
```markdown
- **섹션명**
```
- 볼드 처리 (`**텍스트**`)
- 기능 분야별 분류:
  - 페이지 관리
  - 셀(사진 칸) 조작
  - 도움말
  - 저장 방식 개선
  - 코드 안정성 점검
  - 입력 편의성
  - 버그 수정
  - 배포
  - 기타

### 3. **상세 항목 (소제목)**
```markdown
  - 항목 내용
```
- 2칸 들여쓰기
- 각 항목은 새 줄
- 마침표로 끝남

### 4. **다중 행 설명**
길고 복잡한 설명은 **괄호 내부에** 상세 설명:
```markdown
- 드래그 핸들·삭제 버튼 크기 2배 확대, 교체 버튼을 삭제 버튼 아래 배치
  - 사진 교체 버튼 위치 여러 차례 조정 → 최종적으로 패널 썸네일 hover 버튼으로 확정
```

## 🎯 문체 및 내용

### 액션 동사 사용
- **추가**: "추가", "도입", "신규 기능"
- **수정**: "수정", "개선", "변경", "조정"
- **삭제**: "삭제", "제거"
- **버그 수정**: "버그 수정", "문제 해결"

### 예시
✅ 좋은 예:
```markdown
- 클립보드 붙여넣기(Ctrl+V) 지원 — 이미지 붙여넣으면 마지막 페이지 빈 칸에 자동 추가
- 사진 드래그·확대 시 window 이벤트 리스너가 계속 누적되던 메모리 누수 버그 수정
- 월별 저장 슬롯 기능 추가 — 달마다 독립된 저장 슬롯 생성/전환
```

❌ 피해야 할 문체:
```markdown
- 음... 클립보드 붙여넣기를 했어.
- 그냥 몇 가지를 수정했음.
```

## 🏷️ 심볼 및 이모지 (선택)

자주 쓰이는 심볼:
- `→`: 변경 흐름 ("A → B로 변경")
- `—`: 설명 연결 ("기능명 — 설명")
- `📤`: 내보내기
- `📥`: 불러오기
- `✅`: 완료/추가됨 (선택)

## 🔄 버전 번호 규칙

### 초기 릴리스
```
V1.0 (초기 버전)
```

### 이후 업데이트
- **V1.1 ~ V1.9**: 마이너 업데이트 (기능 추가, 버그 수정)
- **V2.0**: 메이저 업데이트 (구조 변경, 대규모 리팩토링)

### 다중 버전 담기
같은 날짜에 여러 버전을 한 항목으로:
```markdown
## 2026-07-03 (V1.4 ~ V1.9 안정성 개선 및 신규 기능)
```

---

## 📝 작성 순서

1. **버전 헤더** 작성 (날짜 + 버전 + 제목)
2. **섹션** 구분 (기능 영역별)
3. **상세 항목** 나열 (각 항목당 1-2줄)
4. **동사** 명확하게 (추가/수정/삭제)
5. **기술 용어** 포함 (localStorage, IndexedDB, 메모리 누수 등)

---

## 📦 실제 예시

```markdown
## 2026-07-03 (V1.4 ~ V1.9 안정성 개선 및 신규 기능)
- **저장 방식 개선**
  - localStorage 저장 실패(용량 초과) 시 조용히 무시되던 문제 → 경고 토스트 표시
  - 이미지 압축률 조정, 저장 방식을 localStorage → IndexedDB로 전환 (5MB 용량 제한 해제)
  - 월별 저장 슬롯 기능 추가 — 달마다 독립된 저장 슬롯 생성/전환

- **오픈용 헤더 (신규 기능)**
  - 검정 그라데이션 오버레이 스타일의 두 번째 헤더 옵션 최초 추가
  - 그리드 위에 덮어씌우는 2계층 오버레이 구조로 개선

- **코드 안정성 점검**
  - 사진 드래그·확대 시 window 이벤트 리스너가 계속 누적되던 메모리 누수 버그 수정
  - 그리드 빈 공간 드롭 시 이벤트 중복 실행 버그 수정, 죽은 코드 정리

- **기타**
  - 파일명을 megahouse.html → megahouse-matome.html로 변경
```

---

## ✨ 팁

1. **날짜는 실제 작업 완료 날짜**로 기록
2. **버그 수정**은 현상 + 원인 + 해결책 명시
3. **신규 기능**은 "신규 기능" 또는 이모지로 표시
4. **기술 용어**는 정확하게 (예: localStorage, IndexedDB, 메모리 누수)
5. **비정기 배포** 또는 여러 날에 거쳐 작업한 경우, **"배포" 섹션**에 따로 표시
6. **매우 큰 변경사항**은 별도 섹션으로 분리
