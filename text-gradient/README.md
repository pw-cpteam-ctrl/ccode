# 텍스트 그라데이션 이미지 생성기

배지 문구 / 좌우 타이틀(`|` 구분) / 좌우 2열 불릿 리스트를 입력하면, 검정
그라데이션이 깔린 **투명 배경 PNG(1080×1350)** 를 만들어 주는 도구입니다.
빌드/서버 없이 `index.html`을 브라우저로 바로 엽니다.

## 사용법

1. 왼쪽 패널에 배지 문구·타이틀(좌/우)·서브타이틀·불릿 리스트(좌열/우열, 줄바꿈 구분)를 입력한다.
2. 그라데이션 시작점/퍼짐/농도/압축 슬라이더로 하단 그라데이션을 조절한다.
3. **PNG 다운로드 (투명)** 클릭 → `slot{n}.png` 저장 (배경 투명, 1080×1350 고정).
4. 오른쪽 미리보기에서 **원하는 슬롯의 캔버스를 클릭**하면 그 슬롯이 바로
   편집 대상(파란 테두리로 표시)으로 전환된다 — 탭 번호를 찾아 누를 필요 없이
   바둑판 뷰에서 바로 골라 클릭하면 된다.

## 불릿 리스트 (좌/우 2열, 선택 사용)

**기본은 1열(입력칸도 이미지도 폭 전체)**. "2열 사용"을 체크해야만 좌열/우열 두
입력칸이 나타나고, 이미지에도 그대로 2열로 나란히 반영된다 — 캐릭터가 많아 한
줄로 쌓으면 너무 길어질 때만 켜서 좌우로 나눠 담는 용도. 체크를 해제하면 다시
좌열 하나만 폭 전체로 보인다(우열에 입력해 둔 내용은 지워지지 않고 남아있다가,
다시 체크하면 그대로 돌아온다).

- **2열 폭 비율**: 조절 방법이 세 가지이며, 셋 다 완전히 동기화된다(20:80~80:20,
  기본 50:50).
  1. 패널의 **슬라이더**
  2. **좌열/우열 입력창 사이의 회색 손잡이**(⋮)를 마우스로 드래그
  3. **미리보기 이미지의 좌/우 불릿 사이에 뜨는 파란 손잡이**를 마우스로 드래그
     (2열 사용 체크 시에만 나타남)
  한쪽 문구가 길어서 1:1로는 줄바꿈이 어색하게 잘리는 경우, 입력창을 보면서
  조절하고 싶으면 2번, 완성된 이미지를 보면서 조절하고 싶으면 3번을 쓰면 된다.
  이미지 쪽 손잡이는 화면 미리보기에만 있는 편집용 UI라 PNG로 내보낼 때는
  포함되지 않는다.

각 줄은 **앞에 `-`(하이픈)가 있어야 그 줄만 `•` 불릿으로 표시**되고, 마커가 없는
줄은 그냥 일반 텍스트로 표시된다 — 불릿을 켜고 끄는 스위치 대신, 줄 단위로 자유
롭게 섞어 쓸 수 있게 한 것.
```
- 곤 프릭스
- 키르아 조르딕
품절 임박 안내 (마커 없어서 그냥 텍스트로 표시)
```

## 마크다운 일괄입력

여러 슬롯을 한 번에 초안으로 뽑고 싶을 때 사용한다. 두 가지 입력 형식을 지원한다.

**① 볼드 헤더 형식** (월별 신제품 목록처럼 이모지·헤더가 섞인 텍스트를 그대로 붙여넣을 때)

```
**⚔️ 헌터×헌터 | 룩업**
- 곤 프릭스
- 키르아 조르딕

**🏴‍☠️ 원피스 | 메가캣 프로젝트**
- 냥피스 냥! DINER!
```

- **`**좌 | 우**` 볼드 헤더 줄 = 슬롯 1개.** 헤더 앞의 이모지 등 기호는 자동 제거된다.
- **헤더 아래 `-` 항목들은(들여쓰기 없어도) 전부 그 슬롯의 불릿 리스트**로 묶인다 —
  다음 헤더가 나오기 전까지 계속 같은 슬롯에 붙는다.
- `---` 구분선이나 헤더/불릿 형식이 아닌 메모성 줄은 무시된다.

**② 기존 들여쓰기 형식** (붙여넣을 텍스트에 `**볼드**` 헤더가 하나도 없을 때 자동으로 이 방식 사용)

```
- 주술회전 | 메가캣 프로젝트
  - 주술냥코 시부야 사변 (재판)
  - 유지/메구미/노바라
  - 고죠/나나미/스쿠나
- 진격의 거인 | 스케일 피규어
  - 에렌 예거 & 리바이 병장
```

- **최상위 불릿(`-`) = 슬롯 1개.** 텍스트에 `|`가 있으면 좌/우 타이틀로 자동 분리.
- **들여쓴 하위 불릿 = 그 슬롯의 불릿 리스트.** (들여쓰기가 없으면 전부 별도 슬롯으로
  쪼개지니 주의 — 하위 항목을 한 슬롯에 묶으려면 반드시 들여쓸 것)

**공통 사항**
- **배지 문구는 "공통 배지 문구" 입력값이 생성되는 슬롯 전체에 그대로 적용**된다
  (마크다운 자체엔 배지를 넣을 자리가 없음).
- **일괄 적용을 누르면 기존 슬롯은 전부 지워지고 새로 만들어진다** — 잘못 눌렀으면
  Ctrl+Z로 복구.
- 슬롯 개수 제한은 없다. 다만 **100개 초과 시 "계속할까요?" 확인창**이 한 번 뜬다
  (하드 제한 아님, 실수로 대량 텍스트를 잘못 붙여넣었을 때 대비용).
- 그라데이션(시작점/퍼짐/농도/압축) 설정은 마크다운에 없으므로, 일괄 적용 직전에
  보고 있던 슬롯의 그라데이션 값이 새로 생성되는 슬롯 전체에 그대로 유지된다.
- 일괄 생성은 초안 용도이므로, 2단 배치처럼 슬롯마다 다른 세부 조정은 생성 후
  슬롯별로 직접 손본다.

## 배치 · 크기

배지 / 타이틀 / 서브타이틀 / 불릿을 각각 따로 위(Y)/아래로 옮기고 글자 크기를
조절할 수 있다. 슬라이더는 두 그룹으로 나뉜다.

- **개별 조정**: 지금 보고 있는 슬롯 하나에만 적용된다. 슬롯마다 독립적으로 저장
  되므로, 마크다운으로 일괄 생성한 뒤 특정 슬롯만 배치를 다르게 손봐도 다른
  슬롯에는 영향이 없다.
- **전체 적용**: 슬라이더를 움직이면 **모든 슬롯**의 해당 값이 한 번에 같은 값으로
  덮어써진다. 슬롯을 수십 개 만든 뒤 전체 배치를 한 번에 맞출 때 쓴다 — 슬롯마다
  일일이 들어가서 손볼 필요 없이 한 번에 정렬된다.
- **자간**: 타이틀·서브타이틀·불릿 텍스트가 기본 폰트로는 너무 넓어 보여서, 슬라이더
  값(-100~100)을 1000으로 나눈 em 단위로 `letter-spacing`에 적용한다(예: -50 →
  -0.05em, 64px 타이틀 기준 약 -3.2px). **기본값은 -50**로 처음부터 타이트하게 나온다.

**서브타이틀**은 타이틀과 불릿 리스트 사이에 들어가는 한 줄짜리 보조 문구
(예: "2026년 6월 예약판매 시작" 같은 안내성 텍스트)로, 비워두면 화면에 아무것도
나오지 않는다.

## 그라데이션

- **시작점 / 퍼짐 / 농도**: 하단 그라데이션이 시작되는 위치, 퍼지는 폭, 진하기.
- **압축**: 그라데이션이 깔리는 영역 자체를 위에서부터 눌러 좁힌다. 값을 올릴수록
  어두워지는 밴드가 하단에 더 압축되어 몰린다.
- **배지**는 텍스트가 사진에 묻히지 않도록 기본 배경 오퍼시티가 90%로 고정돼
  있고, **타이틀·불릿 텍스트에는 그림자**가 항상 들어가 있어 흰 글자가 밝거나
  복잡한 사진 위에서도 잘 읽힌다.

## 저장 슬롯

- 상단 탭에서 슬롯 전환·추가(＋, 개수 제한 없음)·삭제(×, 슬롯이 2개 이상일 때만) 가능.
- 슬롯 탭 클릭 시 해당 슬롯 위치로 부드럽게 스크롤 이동.
- 모든 입력은 **IndexedDB에 자동 저장**되어 새로고침해도 유지된다.
- **Ctrl+Z**로 실행취소 (최대 30단계, 입력창 포커스 중에는 비활성).
- **개별 다운로드**: 각 슬롯 미리보기 카드 우측 상단의 **"⬇ 다운로드" 버튼**을
  누르면 그 슬롯을 편집 대상으로 전환하지 않고도 바로 그 슬롯만 PNG로 저장된다
  (패널 하단의 "PNG 다운로드" 버튼은 지금 편집 중인 슬롯을 저장한다).

## 미리보기 배치 (가로 일렬 ↔ 바둑판)

스테이지 상단 토글로 두 방식을 오갈 수 있다. **기본값은 바둑판**이며, 창 너비에
따라 한 줄에 보이는 열 개수가 자동으로 바뀐다(인스타/윈도우 탐색기 그리드와
동일한 방식 — 넓히면 더 많이, 좁히면 더 적게). 선택한 모드는 IndexedDB에
저장되어 새로고침해도 유지된다.

## 인디자인 파일목록 생성기

어도비 인디자인 데이터 병합(Data Merge)용 소스 파일을 만드는 유틸리티. 그라데이션
슬롯 기능과는 독립적으로 동작한다.

- 파일명을 한 줄에 하나씩 입력하고(확장자 생략 시 `.png` 자동 추가), 사진 폴더 경로를
  넣고 **유니코드 텍스트로 다운로드**를 누르면 `<@사진1>` 헤더 + 전체 경로 목록이
  UTF-16LE(BOM 포함) 텍스트로 저장된다. 인디자인 데이터 병합 가져오기에 바로 쓸 수 있다.
- **그룹 기능**: `[제목파일명.png]` 줄을 넣으면 그 아래 사진들이 한 그룹으로 묶이고,
  **그룹의 첫 번째 사진에만** 투명제목 이미지 경로가 2번째 열로 붙는다(나머지 사진은
  빈 칸). 인디자인에서 폴더의 맨 첫 장에만 제목을 붙이는 실제 조판 방식과 일치하도록
  맞춘 것 — 그룹 사용 시엔 "투명제목 폴더 경로"도 함께 입력해야 한다.
  ```
  [바쿠고_제목.png]
  바쿠고1
  바쿠고2
  [나루토_제목.png]
  나루토1
  나루토2
  ```
- **폴더 자동 인식** (크롬/엣지 전용, File System Access API): 캐릭터별로 하위 폴더가
  나뉜 상위 폴더를 선택하면, 폴더마다 파일명에 "표지"가 들어간 사진(없으면 이름순
  첫 번째 사진)을 그룹 첫 장으로 자동 배치하고, 제목 파일명은 `폴더명_제목.png`
  규칙으로 자동 생성해 파일명 목록 칸을 채워준다 — 수십 장을 일일이 타이핑할 필요가
  없어진다. 브라우저 보안상 실제 경로는 읽을 수 없으므로, 다운로드용 "사진 폴더 경로"
  칸은 이 기능과 별개로 계속 직접 입력해야 한다.

## 구현 메모

- 화면 미리보기는 `.canvas`(1080×1350 원본) 에 `transform:scale(0.3)`을 걸어
  `.canvas-clip`(축소 후 실제 크기) 안에 담아 보여준다. PNG로 내보낼 때는
  이 transform을 잠깐 제거하고 원본 크기 그대로 캡처한 뒤 복원한다 — 이 처리를
  빠뜨리면 캡처 결과가 캔버스 좌상단 한쪽에 쪼그라들어 찍히니 주의.
